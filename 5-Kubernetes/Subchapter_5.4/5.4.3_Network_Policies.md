# 5.4.3 Network Policies: Pod-to-Pod Isolation

#### Why Network Policies Matter

By default, Kubernetes allows **all traffic** between all pods. This is convenient but insecure. **Network Policies** provide firewall rules at the pod level, enabling zero-trust networking:

* **Isolate environments** (production vs staging)

* **Secure multi-tenant clusters** (prevent tenant A from accessing tenant B)

* **Restrict database access** (only API pods can reach database)

* **Compliance** (PCI, HIPAA, SOC2)

This note covers NetworkPolicy resources. Note 5.4.1 covered Services; note 5.4.2 covered Ingress; note 5.4.4 is the subchapter review.

**Backward references:** Pod labels from 5.3.1 (selectors); Namespaces from 5.3.1; Network fundamentals from Module 2 (firewall rules, ingress/egress).

## Part 1: Network Policy Prerequisites

### CNI Requirements

Network Policies require a CNI plugin that supports them:

| CNI             | Network Policy Support                         |
| --------------- | ---------------------------------------------- |
| **Calico**      | Full support (default for many clusters)       |
| **Cilium**      | Full support + L7 policies                     |
| **Weave**       | Full support                                   |
| **Flannel**     | **No support** (requires additional component) |
| **Antrea**      | Full support                                   |
| **Kube-router** | Full support                                   |

```bash
# Check if your CNI supports network policies
kubectl get pods -n kube-system | grep -E "calico|cilium|weave|antrea"

# If using Flannel, install Calico for network policies only
kubectl apply -f https://raw.githubusercontent.com/projectcalico/calico/v3.27/manifests/canal.yaml
```

### Policy Types

| Type        | Controls                   |
| ----------- | -------------------------- |
| **Ingress** | Incoming traffic to pods   |
| **Egress**  | Outgoing traffic from pods |

## Part 2: Basic Network Policy Examples

### Deny All Ingress Traffic (Default Deny)

```yaml
# deny-all-ingress.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all-ingress
spec:
  podSelector: {}  # Selects all pods in namespace
  policyTypes:
  - Ingress
  # No ingress rules = deny all ingress
```

### Allow All Ingress Traffic

```yaml
# allow-all-ingress.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-all-ingress
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - {}  # Empty rule = allow all ingress
```

### Allow Ingress from Specific Pod

```yaml
# allow-from-frontend.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-from-frontend
spec:
  podSelector:
    matchLabels:
      app: backend
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: frontend
    ports:
    - protocol: TCP
      port: 8080
```

**Effect:** Only pods with label `app: frontend` can access pods with label `app: backend` on port 8080.

## Part 3: Ingress Rules (Incoming Traffic)

### Allow from Specific Namespace

```yaml
# allow-from-monitoring-ns.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-from-monitoring
spec:
  podSelector:
    matchLabels:
      app: app
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          kubernetes.io/metadata.name: monitoring
    ports:
    - protocol: TCP
      port: 9090
```

### Allow from Specific Pod in Specific Namespace

```yaml
# allow-from-prometheus.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-from-prometheus
spec:
  podSelector:
    matchLabels:
      app: app
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: monitoring
      podSelector:
        matchLabels:
          app: prometheus
    ports:
    - protocol: TCP
      port: 9090
```

### Allow from IP Block (CIDR)

```yaml
# allow-from-cidr.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-from-cidr
spec:
  podSelector:
    matchLabels:
      app: database
  policyTypes:
  - Ingress
  ingress:
  - from:
    - ipBlock:
        cidr: 10.0.0.0/8
        except:
        - 10.0.1.0/24
    ports:
    - protocol: TCP
      port: 5432
```

## Part 4: Egress Rules (Outgoing Traffic)

### Allow Egress to Specific Service

```yaml
# allow-egress-to-db.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-egress-to-db
spec:
  podSelector:
    matchLabels:
      app: api
  policyTypes:
  - Egress
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: database
    ports:
    - protocol: TCP
      port: 5432
  - to:
    - namespaceSelector: {}  # Allow DNS (kube-dns)
      podSelector:
        matchLabels:
          k8s-app: kube-dns
    ports:
    - protocol: UDP
      port: 53
    - protocol: TCP
      port: 53
```

### Allow Egress to External IP

```yaml
# allow-egress-to-external.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-egress-to-external
spec:
  podSelector:
    matchLabels:
      app: app
  policyTypes:
  - Egress
  egress:
  - to:
    - ipBlock:
        cidr: 8.8.8.8/32
    ports:
    - protocol: UDP
      port: 53
  - to:
    - ipBlock:
        cidr: 0.0.0.0/0
    ports:
    - protocol: TCP
      port: 443
```

### Deny All Egress (Default Deny)

```yaml
# deny-all-egress.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all-egress
spec:
  podSelector: {}
  policyTypes:
  - Egress
  # No egress rules = deny all egress
```

## Part 5: Complete Isolation Example

```yaml
# isolate-database.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: database-isolation
  namespace: production
spec:
  podSelector:
    matchLabels:
      app: database
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: api
    - namespaceSelector:
        matchLabels:
          kubernetes.io/metadata.name: monitoring
      podSelector:
        matchLabels:
          app: prometheus
    ports:
    - protocol: TCP
      port: 5432
  egress:
  - to:
    - podSelector: {}  # Allow to other pods in same namespace? No, this allows all
    # Actually, for complete isolation, don't include this
    # Instead, only allow to kube-dns and backups
    - namespaceSelector: {}
      podSelector:
        matchLabels:
          k8s-app: kube-dns
    ports:
    - protocol: UDP
      port: 53
  - to:
    - ipBlock:
        cidr: 10.0.0.0/8
    ports:
    - protocol: TCP
      port: 443
```

## Part 7: Network Policy Troubleshooting

### Check if Network Policy is Enforced

```bash
# List network policies
kubectl get networkpolicies
# or
kubectl get netpol

# Describe policy
kubectl describe netpol allow-from-frontend

# Check if CNI supports policies
kubectl get pods -n kube-system | grep -E "calico|cilium|weave|antrea"
```

### Testing Network Policies

```bash
# Create test pods in different namespaces
kubectl run test-frontend --image=nginx --labels="app=frontend"
kubectl run test-backend --image=nginx --labels="app=backend"
kubectl run test-other --image=nginx --labels="app=other"

# Get pod IPs
kubectl get pods -o wide

# Test connectivity from frontend to backend
kubectl exec test-frontend -- curl http://<backend-ip>:80

# Test from other pod (should fail if policy denies)
kubectl exec test-other -- curl http://<backend-ip>:80

# Test from backend to frontend (egress vs ingress)
kubectl exec test-backend -- curl http://<frontend-ip>:80
```

### Debugging with Temporary Pods

```bash
# Run a debug pod with networking tools
kubectl run debug --rm -it --image=nicolaka/netshoot -- /bin/bash

# Inside debug pod:
curl http://target-service:80
tcpdump -i eth0
nslookup google.com
```

### Common Network Policy Issues

| Issue                      | Likely Cause                            | Fix                                      |
| -------------------------- | --------------------------------------- | ---------------------------------------- |
| Policy not working         | CNI doesn't support policies            | Install Calico/Cilium                    |
| All traffic blocked        | Default deny policy without allow rules | Add allow rules for DNS, health checks   |
| Cross-namespace fails      | Namespace selector incorrect            | Verify namespace labels                  |
| Egress to internet blocked | No egress rule for external IPs         | Add `ipBlock` for 0.0.0.0/0              |
| DNS not working            | Port 53 UDP not allowed                 | Add rule for kube-dns on port 53 UDP/TCP |

## Part 8: Real-World Policy Examples

### Example 1: Three-Tier Application

```yaml
# Tier 1: Web (public) – Allow ingress from anywhere
# Tier 2: API – Allow ingress only from Web
# Tier 3: Database – Allow ingress only from API

---
# allow-ingress-to-web.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-web-ingress
spec:
  podSelector:
    matchLabels:
      tier: web
  policyTypes:
  - Ingress
  ingress:
  - {}

---
# allow-ingress-to-api.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-api-ingress
spec:
  podSelector:
    matchLabels:
      tier: api
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          tier: web
    ports:
    - protocol: TCP
      port: 8080

---
# allow-ingress-to-db.yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-db-ingress
spec:
  podSelector:
    matchLabels:
      tier: db
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          tier: api
    ports:
    - protocol: TCP
      port: 5432
```

### Example 2: Multi-Tenant Cluster

```yaml
# tenant-isolation.yaml
# Allow traffic only within same tenant namespace
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: tenant-isolation
  namespace: tenant-a
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          tenant: tenant-a
  egress:
  - to:
    - namespaceSelector:
        matchLabels:
          tenant: tenant-a
    # Allow DNS
    - namespaceSelector: {}
      podSelector:
        matchLabels:
          k8s-app: kube-dns
    ports:
    - protocol: UDP
      port: 53
```

### Example 3: Allow Health Checks


## Quick Task: Implement Network Policies

*Create a secure environment with network policies.*

1. Create a namespace `secure` with labels.
2. Create two deployments: `frontend` and `backend`.
3. Apply a default deny all ingress policy.
4. Create a policy allowing `frontend` to access `backend` on port 80.
5. Test connectivity from both pods.

> **Ready Solution:**
>
> ```bash
> # Task 1
> kubectl create ns secure
> kubectl label ns secure name=secure
>
> # Task 2
> kubectl run frontend --image=nginx --port=80 -n secure --labels="app=frontend"
> kubectl run backend --image=nginx --port=80 -n secure --labels="app=backend"
>
> # Task 3
> cat << EOF | kubectl apply -f -
> apiVersion: networking.k8s.io/v1
> kind: NetworkPolicy
> metadata:
>   name: deny-all-ingress
>   namespace: secure
> spec:
>   podSelector: {}
>   policyTypes:
>   - Ingress
> EOF
>
> # Task 4
> cat << EOF | kubectl apply -f -
> apiVersion: networking.k8s.io/v1
> kind: NetworkPolicy
> metadata:
>   name: allow-frontend-to-backend
>   namespace: secure
> spec:
>   podSelector:
>     matchLabels:
>       app: backend
>   policyTypes:
>   - Ingress
>   ingress:
>   - from:
>     - podSelector:
>         matchLabels:
>           app: frontend
>     ports:
>     - protocol: TCP
>       port: 80
> EOF
>
> # Task 5
> FRONTEND_IP=$(kubectl get pod frontend -n secure -o jsonpath='{.status.podIP}')
> BACKEND_IP=$(kubectl get pod backend -n secure -o jsonpath='{.status.podIP}')
>
> # From frontend (should succeed)
> kubectl exec frontend -n secure -- curl -s http://$BACKEND_IP
>
> # From backend (should fail - no egress rule)
> kubectl exec backend -n secure -- curl -s http://$FRONTEND_IP
> ```

## Summary Table: Network Policy Rules

| Rule Type             | Selector | Purpose                                         |
| --------------------- | -------- | ----------------------------------------------- |
| **podSelector**       | Labels   | Select pods in same namespace                   |
| **namespaceSelector** | Labels   | Select all pods in namespace                    |
| **ipBlock**           | CIDR     | External IP ranges                              |
| **Combined**          | Both     | Pods in specific namespace with specific labels |

### Policy Types

| Type      | Controls         |
| --------- | ---------------- |
| `Ingress` | Incoming traffic |
| `Egress`  | Outgoing traffic |
| Both      | Both directions  |

### Common Ports to Allow

| Port  | Protocol | Purpose                |
| ----- | -------- | ---------------------- |
| 53    | UDP/TCP  | DNS (kube-dns/CoreDNS) |
| 8472  | UDP      | Flannel VXLAN          |
| 6443  | TCP      | API server             |
| 10250 | TCP      | Kubelet metrics        |
| 10255 | TCP      | Kubelet readonly       |

### Network Policy Commands

| Command                                                            | Purpose               |
| ------------------------------------------------------------------ | --------------------- |
| `kubectl get netpol`                                               | List network policies |
| `kubectl describe netpol NAME`                                     | Show policy details   |
| `kubectl delete netpol NAME`                                       | Delete policy         |
| `kubectl run test --image=nicolaka/netshoot --rm -it -- /bin/bash` | Debug pod             |
