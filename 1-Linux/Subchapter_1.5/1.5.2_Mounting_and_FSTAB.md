## 1.5.2 Mounting and FSTAB: Attaching Storage to the Filesystem Tree

#### Why Mounting Matters

In Linux, everything lives under a single directory tree starting at `/`. When you add a new disk or partition, you must **mount** it – attach it to an empty directory (mount point) somewhere in that tree. Understanding mounting is essential for:

* Adding new storage volumes to servers

* Troubleshooting "filesystem full" errors

* Understanding container storage (volumes, bind mounts in Docker)

* Configuring automatic mounts at boot via `/etc/fstab`

***

## Part 1: The Mount Concept

### Before Mounting: Raw Partition

A partition with a filesystem exists but is not accessible. Think of it as a formatted hard drive sitting on a shelf – the data is there, but you cannot read it until you plug it in.

```bash
# Partition exists and has a filesystem
sudo blkid /dev/sdb1
# /dev/sdb1: UUID="abc123" TYPE="ext4"

# But it's not part of the directory tree
ls /mnt/data
# ls: cannot access '/mnt/data': No such file or directory
```

### After Mounting: Attached to Directory

Mounting attaches the filesystem to an empty directory. After mounting, the directory becomes the **root of that filesystem**.

```bash
# Create mount point (empty directory)
sudo mkdir -p /mnt/data

# Mount the partition
sudo mount /dev/sdb1 /mnt/data

# Now /mnt/data shows the contents of /dev/sdb1
ls /mnt/data
# lost+found  (empty ext4 filesystem has this directory)
```

## Part 2: The `mount` Command

### Basic Mount Operations

```bash
# Mount a partition (requires device and mount point)
sudo mount /dev/sdb1 /mnt/data

# Mount with specific filesystem type (usually auto-detected)
sudo mount -t ext4 /dev/sdb1 /mnt/data

# Mount with read-only access
sudo mount -o ro /dev/sdb1 /mnt/data

# Mount with noatime (don't update access time – improves performance)
sudo mount -o noatime /dev/sdb1 /mnt/data

# Mount all filesystems listed in /etc/fstab
sudo mount -a

# Remount (change mount options, e.g., read-write to read-only)
sudo mount -o remount,ro /mnt/data

# Mount by UUID (preferred for persistence)
sudo mount UUID="abc123-def456-..." /mnt/data

# Mount by label
sudo mount LABEL="mydata" /mnt/data
```

### Understanding `mount` Options

| Option     | Meaning                                            | Use Case                                                     |
| ---------- | -------------------------------------------------- | ------------------------------------------------------------ |
| `ro`       | Read-only                                          | Recovery, forensic analysis, shared filesystems              |
| `rw`       | Read-write (default)                               | Normal operation                                             |
| `noatime`  | Don't update access timestamps                     | Performance (SSDs, databases)                                |
| `relatime` | Update atime only if older than mtime              | Default in modern kernels                                    |
| `noexec`   | Prevent execution of binaries                      | Security (e.g., `/tmp` should be noexec)                     |
| `nosuid`   | Ignore SUID/SGID bits                              | Security (prevent privilege escalation from removable media) |
| `nodev`    | Ignore device files                                | Security (prevent device access from USB drives)             |
| `sync`     | Write changes immediately (not buffered)           | Very slow, only for removable media                          |
| `async`    | Write asynchronously (default, better performance) | Normal operation                                             |

**Example: Secure mount for** **`/tmp`** **(if** **`/tmp`** **is a separate partition):**

```bash
sudo mount -o rw,nosuid,nodev,noexec,relatime /dev/sdb1 /tmp
```

### Viewing Mounted Filesystems

```bash
# List all mounted filesystems
mount
# Output: /dev/sda2 on / type ext4 (rw,relatime)
#         /dev/sda1 on /boot type ext4 (rw,relatime)
#         /dev/sdb1 on /mnt/data type ext4 (rw,noatime)

# More readable format
findmnt
# Output as tree:
# TARGET    SOURCE     FSTYPE OPTIONS
# /         /dev/sda2  ext4   rw,relatime
# ├─/boot   /dev/sda1  ext4   rw,relatime
# └─/mnt/data /dev/sdb1 ext4   rw,noatime

# Show only specific mount point
findmnt /mnt/data

# Show filesystem type
df -T
# Filesystem     Type      Size  Used Avail Use% Mounted on
# /dev/sdb1      ext4       50G  1.2G   46G   3% /mnt/data
```

### Unmounting

```bash
# Unmount by mount point
sudo umount /mnt/data

# Unmount by device (less common)
sudo umount /dev/sdb1

# Lazy unmount (detach now, cleanup when no longer busy)
sudo umount -l /mnt/data

# Force unmount (dangerous – may cause data loss)
sudo umount -f /mnt/data
```

**"Target is busy" error:** Cannot unmount if a process is using the mount point.

```bash
# Find processes using the mount point
sudo lsof /mnt/data
sudo fuser -v /mnt/data

# Kill processes using the mount point (be careful)
sudo fuser -km /mnt/data

# Then unmount
sudo umount /mnt/data
```

## Part 3: The `/etc/fstab` File – Automatic Mounting at Boot

`/etc/fstab` (File System Table) tells the system which filesystems to mount at boot and with what options.

### FSTAB Format

Each line has **6 fields** (whitespace-separated, can be spaces or tabs):

```
[Device]   [Mount Point]   [FS Type]   [Options]   [Dump]   [Pass]
```

**Example** **`/etc/fstab`:**

```bash
# /etc/fstab: static file system information.
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

# Root filesystem
UUID=a1b2c3d4-... /               ext4    errors=remount-ro 0       1

# Boot partition
UUID=e5f6g7h8-... /boot           ext4    defaults          0       2

# Swap
UUID=i9j0k1l2-... none            swap    sw                0       0

# Data partition (by label)
LABEL=mydata      /mnt/data       ext4    defaults,noatime  0       2

# NFS share (network filesystem)
server:/exports   /mnt/nfs        nfs     defaults          0       0

# tmpfs (RAM disk)
tmpfs             /tmp            tmpfs   defaults,noexec,nosuid,nodev 0 0
```

### Field-by-Field Breakdown

| Field           | Purpose                                    | Common Values                                                  |
| --------------- | ------------------------------------------ | -------------------------------------------------------------- |
| **Device**      | What to mount                              | `UUID=...`, `LABEL=...`, `/dev/sda1`, `server:/path` (NFS)     |
| **Mount Point** | Where to attach                            | `/. `/boot`, `/home`, `/mnt/data`, `none\` (for swap)          |
| **FS Type**     | Filesystem type                            | `ext4`, `xfs`, `btrfs`, `swap`, `nfs`, `tmpfs`, `auto`         |
| **Options**     | Mount options (comma-separated, no spaces) | `defaults`, `ro`, `noatime`, `noexec`                          |
| **Dump**        | Backup flag (0 or 1)                       | `0` (disable dump), `1` (enable – rare)                        |
| **Pass**        | fsck order (0,1,2)                         | `0` (no check), `1` (root filesystem), `2` (other filesystems) |

### Common Mount Options for FSTAB

| Option              | Meaning                                               | When to Use                   |
| ------------------- | ----------------------------------------------------- | ----------------------------- |
| `defaults`          | `rw,suid,dev,exec,auto,nouser,async`                  | Most filesystems              |
| `ro`                | Read-only                                             | Recovery, shared data         |
| `noatime`           | Don't update access time                              | Performance (SSDs, databases) |
| `noexec`            | No binary execution                                   | `/tmp`, `/var/tmp`            |
| `nosuid`            | No SUID/SGID                                          | `/tmp`, removable drives      |
| `nodev`             | No device files                                       | `/tmp`, `/var`                |
| `auto` / `noauto`   | Mount at boot (`auto`) or not (`noauto`)              | `noauto` for optional drives  |
| `user` / `nouser`   | Allow users to mount (`user`) or root only (`nouser`) | `user` for removable media    |
| `exec` / `noexec`   | Allow binary execution                                | Security hardening            |
| `relatime`          | Update atime only if older than mtime                 | Default, good balance         |
| `errors=remount-ro` | Remount read-only on error                            | Root filesystem               |
| `_netdev`           | Network filesystem (wait for network)                 | NFS, SMB, iSCSI               |

### Understanding Dump and Pass

**Dump (field 5):**

* `0` – Do not back up with `dump` command (rarely used)

* `1` – Back up (almost never set; `dump` is obsolete)

**Pass (field 6) – fsck order:**

* `0` – Do not check at boot

* `1` – Check first (only for root filesystem `/`)

* `2` – Check after root (all other filesystems)

**Why** **`pass`** **matters:** Root must be checked before other filesystems because other mounts may depend on it. Set root to `1`, everything else to `2` (or `0` to skip).

### UUID vs Device Names: Why UUID is Better

| Method      | Example             | Problem                                                  |
| ----------- | ------------------- | -------------------------------------------------------- |
| Device name | `/dev/sda1`         | Device names can change (add disk → `sda` becomes `sdb`) |
| UUID        | `UUID=a1b2c3d4-...` | Unique, persistent, never changes                        |
| Label       | `LABEL=mydata`      | Persistent but must be unique (user sets)                |

**Best practice:** Always use **UUID** in `/etc/fstab` for local filesystems. Labels for removable media.

```bash
# Get UUIDs for all filesystems
sudo blkid

# Output:
# /dev/sda1: UUID="a1b2c3d4-..." TYPE="ext4" PARTUUID="..."
# /dev/sda2: UUID="e5f6g7h8-..." TYPE="swap"

# Use in fstab:
UUID=a1b2c3d4-... / ext4 defaults 0 1
```

## Part 4: Testing and Validating FSTAB

### Before Rebooting: Test FSTAB Entries

**Never rely on "it will work at boot." Test first.**

```bash
# Mount all entries in fstab (skips already mounted)
sudo mount -a

# Check exit code
echo $?
# 0 = success, non-zero = error

# If error, check what failed
sudo mount -a -v  # verbose

# Test specific fstab entry (mount by mount point)
sudo mount /mnt/data   # Uses fstab options

# Verify fstab syntax and semantics (modern systems)
findmnt --verify
```

**Why `findmnt --verify` helps:** It checks whether `/etc/fstab` entries are structurally valid and whether mount points, options, and referenced devices look sensible before you reboot. On modern `systemd`-based distributions, fstab entries are translated into mount units during boot, so catching mistakes early reduces emergency-shell surprises.

### Common FSTAB Mistakes and Fixes

| Mistake               | Symptom                           | Fix                                   |
| --------------------- | --------------------------------- | ------------------------------------- |
| Missing UUID          | Boot drops to emergency shell     | Boot from rescue media, correct fstab |
| Wrong filesystem type | `mount: unknown filesystem type`  | Check with `blkid`, correct type      |
| Missing mount point   | `mount point does not exist`      | Create directory (`mkdir -p`)         |
| Incorrect options     | Mount succeeds but wrong behavior | Remount with correct options          |
| Swap with wrong pass  | `swapon: cannot find the device`  | Set `pass` to `0` for swap            |

### Testing FSTAB Without Rebooting

```bash
# Method 1: Unmount and remount with fstab options
sudo umount /mnt/data
sudo mount /mnt/data   # Reads from fstab

# Method 2: Use mount with -o (override)
sudo mount -o remount,rw /mnt/data

# Method 3: Simulate boot (using systemd)
sudo systemctl daemon-reload
sudo systemctl restart local-fs.target
```

## Part 5: Special FSTAB Entries

### Swap Partition

```bash
# /etc/fstab entry for swap
UUID=e5f6g7h8-... none swap sw 0 0

# Enable swap after adding to fstab
sudo swapon -a

# Check swap
sudo swapon --show
# NAME      TYPE SIZE USED PRIO
# /dev/sda2 partition 4G 0B -2
```

### tmpfs (RAM Disk)

```bash
# Mount 2GB RAM disk at /ramcache
tmpfs /ramcache tmpfs size=2G,defaults,noatime 0 0

# Mount /tmp to RAM (fast but limited space)
tmpfs /tmp tmpfs defaults,noexec,nosuid,nodev,size=2G 0 0
```

**Use cases for tmpfs:**

* `/tmp` – Temporary files (fast, cleared on reboot)

* `/run` – Runtime state (already tmpfs)

* `/dev/shm` – Shared memory (already tmpfs)

* Application caches that don't need persistence

### Bind Mounts (Mount Directory Elsewhere)

A bind mount makes a directory appear in another location without copying.

```bash
# FSTAB entry for bind mount
/var/log /mnt/logs none bind 0 0

# Or with options
/var/log /mnt/logs none bind,ro 0 0  # Read-only bind
```

**Use case:** Make a subdirectory accessible elsewhere, or mount a directory read-only for a container.

## Part 6: Troubleshooting Mount Issues

### Problem 1: "Structure needs cleaning"

```bash
# Filesystem corruption
sudo umount /dev/sdb1
sudo fsck.ext4 -f /dev/sdb1
sudo mount /dev/sdb1 /mnt/data
```

### Problem 2: "Can't find UUID"

```bash
# Verify UUID exists
sudo blkid | grep "a1b2c3d4"
# If not found, UUID is wrong – correct it in fstab

# Regenerate UUID (rare)
sudo tune2fs -U random /dev/sdb1
```

### Problem 3: "Mount point is not a directory"

```bash
# Mount point exists but is a file
ls -l /mnt/data
# -rw-r--r-- ... (file, not directory)

# Fix: remove file, create directory
sudo rm /mnt/data
sudo mkdir -p /mnt/data
```

### Problem 4: Boot drops to emergency shell

**Symptom:** System boots to "Give root password for maintenance" or emergency shell.

**Cause:** FSTAB has error (missing UUID, wrong device, etc.).

**Recovery:**

```bash
# At emergency shell (root filesystem mounted read-only)
mount -o remount,rw /
vi /etc/fstab   # Fix the error
mount -a        # Test all mounts
exit            # Continue boot
```

## Quick Task: Add a Persistent Mount

*Using the partition from 1.5.1 (or a new loop device), add it to* *`/etc/fstab`* *and test.*

1. Create a partition `/dev/sdb1` (or `/dev/loop0p1`) with ext4 filesystem.
2. Create a mount point `/mnt/persist`.
3. Get the UUID of the partition using `blkid`.
4. Add an entry to `/etc/fstab` for this partition with `defaults,noatime` options, `dump=0`, `pass=2`.
5. Test mounting with `sudo mount /mnt/persist` (should use fstab).
6. Verify it's mounted with `findmnt /mnt/persist`.
7. Unmount and test `sudo mount -a` (should mount all including your new entry).

> **Ready Solution:**
>
> ```bash
> # Task 1-2 (assume /dev/sdb1 exists from previous lab)
> sudo mkdir -p /mnt/persist
>
> # Task 3
> sudo blkid /dev/sdb1
> # Output: /dev/sdb1: UUID="abc123-def456-..." TYPE="ext4"
> # Copy the UUID (without quotes)
>
> # Task 4 (backup fstab first)
> sudo cp /etc/fstab /etc/fstab.backup
> echo "UUID=abc123-def456-... /mnt/persist ext4 defaults,noatime 0 2" | sudo tee -a /etc/fstab
>
> # Task 5
> sudo mount /mnt/persist
>
> # Task 6
> findmnt /mnt/persist
> # TARGET       SOURCE     FSTYPE OPTIONS
> # /mnt/persist /dev/sdb1  ext4   rw,noatime
>
> # Task 7
> sudo umount /mnt/persist
> sudo mount -a
> findmnt /mnt/persist   # Should be mounted again
>
> # Optional: Test boot (if in VM, reboot)
> # sudo reboot
> ```

## Summary Table: Mount and FSTAB Commands

| Command            | Purpose                    | Example                              |
| ------------------ | -------------------------- | ------------------------------------ |
| `mount`            | Mount filesystem           | `sudo mount /dev/sdb1 /mnt/data`     |
| `mount -a`         | Mount all fstab entries    | `sudo mount -a`                      |
| `mount -o remount` | Change mount options       | `sudo mount -o remount,ro /mnt/data` |
| `umount`           | Unmount                    | `sudo umount /mnt/data`              |
| `findmnt`          | Show mount tree            | `findmnt /mnt/data`                  |
| `df -h`            | Show disk usage            | `df -h /mnt/data`                    |
| `blkid`            | Show UUIDs                 | `sudo blkid /dev/sdb1`               |
| `lsblk -f`         | Show filesystem info       | `lsblk -f`                           |
| `swapon -a`        | Enable all swap            | `sudo swapon -a`                     |
| `fuser`            | Find processes using mount | `sudo fuser -v /mnt/data`            |
| `findmnt --verify` | Validate fstab entries     | `findmnt --verify`                   |

### FSTAB Field Reference

| Field | Purpose           | Examples                              |
| ----- | ----------------- | ------------------------------------- |
| 1     | Device            | `UUID=...`, `LABEL=...`, `/dev/sda1`  |
| 2     | Mount point       | `/`, `/boot`, `/home`, `/mnt/data`    |
| 3     | Filesystem type   | `ext4`, `xfs`, `swap`, `tmpfs`, `nfs` |
| 4     | Options           | `defaults`, `ro`, `noatime`, `noexec` |
| 5     | Dump              | `0` (almost always)                   |
| 6     | Pass (fsck order) | `0` (skip), `1` (root), `2` (others)  |

### Common FSTAB Entries Cheatsheet

| Purpose         | Entry                                              |
| --------------- | -------------------------------------------------- |
| Root filesystem | `UUID=... / ext4 errors=remount-ro 0 1`            |
| Boot partition  | `UUID=... /boot ext4 defaults 0 2`                 |
| Data partition  | `UUID=... /data ext4 defaults,noatime 0 2`         |
| Swap            | `UUID=... none swap sw 0 0`                        |
| tmpfs RAM disk  | `tmpfs /ramdisk tmpfs size=2G,defaults 0 0`        |
| Bind mount      | `/source/dir /target/dir none bind 0 0`            |
| NFS share       | `server:/export /mnt/nfs nfs defaults,_netdev 0 0` |
