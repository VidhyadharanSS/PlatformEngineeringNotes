## 1.7.2 DPKG and APT: Debian Family Package Management

#### Why DPKG/APT Matters

The Debian family (Debian, Ubuntu, Linux Mint, Pop!\_OS) dominates cloud computing – Ubuntu is the most popular Linux distribution on AWS, Azure, and GCP. Understanding Debian package management is essential for platform engineers working in cloud-native environments.

This note covers **DPKG** (low-level Debian package manager) and **APT** (Advanced Package Tool – high-level dependency resolver). Note 1.7.1 covered the RHEL family; note 1.7.3 covers compiling from source.

***

## Part 1: DPKG – The Low-Level Package Manager

**DPKG** (Debian Package) is the foundation. It installs, removes, and queries `.deb` packages but **does not resolve dependencies** – that's APT's job.

### DPKG File Naming Convention

```
package-name_version-architecture.deb
```

**Example:** `nginx_1.18.0-0ubuntu1_amd64.deb`

| Component    | Example           | Meaning                                                      |
| ------------ | ----------------- | ------------------------------------------------------------ |
| Package name | `nginx`           | Software name                                                |
| Version      | `1.18.0-0ubuntu1` | Upstream version + Debian revision                           |
| Architecture | `amd64`           | `amd64` (x86\_64), `arm64`, `all` (architecture-independent) |

### Basic DPKG Commands

```bash
# Install a local .deb file (no dependency resolution)
sudo dpkg -i package.deb
# -i = install

# Remove package (but keep configuration files)
sudo dpkg -r package-name
# -r = remove

# Purge package (remove configuration files too)
sudo dpkg -P package-name
# -P = purge

# List installed packages
dpkg -l
# -l = list

# List specific package status
dpkg -l | grep nginx

# List files installed by a package
dpkg -L package-name

# Find which package owns a file
dpkg -S /path/to/file

# Show package information
dpkg -s package-name

# Show package status (installed, config-files, etc.)
dpkg -s nginx | grep Status

# Reconfigure an installed package (interactive)
sudo dpkg-reconfigure package-name

# Extract .deb contents without installing
dpkg -x package.deb /tmp/extract/

# Check if package is broken
dpkg -C package-name
```

### DPKG Status Codes (from `dpkg -l`)

The first three characters of `dpkg -l` output show desired and actual status:

```
ii  nginx  1.18.0  amd64  web server
```

| Position | Meaning        | Codes                                                                          |
| -------- | -------------- | ------------------------------------------------------------------------------ |
| 1        | Desired action | `i` (install), `r` (remove), `p` (purge), `h` (hold)                           |
| 2        | Current status | `i` (installed), `c` (config-files remain), `r` (removed), `n` (not installed) |
| 3        | Error flag     | (blank = OK), `R` (reinstall required)                                         |

**Common statuses:**

* `ii` – Installed (good)

* `rc` – Removed, but config files remain

* `rn` – Removal requested, not installed

* `iU` – Unpacked (not configured)

### DPKG Database

DPKG maintains its database in `/var/lib/dpkg/`. If corrupted, recovery is possible.

```bash
# Reconfigure all partially installed packages
sudo dpkg --configure -a

# Force remove a broken package (if normal removal fails)
sudo dpkg --remove --force-remove-reinstreq package-name

# Force install over conflicts (dangerous)
sudo dpkg -i --force-overwrite package.deb
```

## Part 2: APT – The High-Level Package Manager

**APT** (Advanced Package Tool) handles dependency resolution, repository management, and automatic upgrades.

### APT vs APT-GET vs APT-CACHE

Modern Debian/Ubuntu provides `apt` as a unified command. Legacy commands still work:

| Legacy Command         | Purpose                      | Modern `apt` Equivalent |
| ---------------------- | ---------------------------- | ----------------------- |
| `apt-get install`      | Install packages             | `apt install`           |
| `apt-get remove`       | Remove packages              | `apt remove`            |
| `apt-get update`       | Update package lists         | `apt update`            |
| `apt-get upgrade`      | Upgrade all packages         | `apt upgrade`           |
| `apt-get dist-upgrade` | Smart upgrade (handles deps) | `apt full-upgrade`      |
| `apt-cache search`     | Search packages              | `apt search`            |
| `apt-cache show`       | Package info                 | `apt show`              |
| `apt-cache policy`     | Package version info         | `apt policy`            |

**Use** **`apt`** **for interactive use,** **`apt-get`** **for scripts** (output is stable).

### Basic APT Commands

```bash
# Update package lists (run before install/upgrade)
sudo apt update

# Upgrade all packages (safe upgrade)
sudo apt upgrade

# Smart upgrade (add/remove packages if needed)
sudo apt full-upgrade

# Install package
sudo apt install nginx

# Install with specific version
sudo apt install nginx=1.18.0-0ubuntu1

# Reinstall package
sudo apt install --reinstall nginx

# Remove package (keep config files)
sudo apt remove nginx

# Purge package (remove config files)
sudo apt purge nginx

# Remove unused dependencies
sudo apt autoremove

# Clean downloaded .deb files
sudo apt clean

# Clean old downloaded .deb files (keeps recent)
sudo apt autoclean

# Search for package
apt search nginx

# Show package information
apt show nginx

# Show package policy (versions, sources)
apt policy nginx

# List installed packages
apt list --installed

# List upgradable packages
apt list --upgradable

# List all available versions
apt list -a nginx

# Show dependencies
apt depends nginx

# Show reverse dependencies (what depends on this)
apt rdepends nginx

# Download package without installing
apt download nginx
```

### APT vs DNF (RHEL Family) – Side by Side

| Operation            | Debian/Ubuntu (APT)    | RHEL/Rocky (DNF)     |
| -------------------- | ---------------------- | -------------------- |
| Update package lists | `apt update`           | `dnf makecache`      |
| Install package      | `apt install nginx`    | `dnf install nginx`  |
| Remove package       | `apt remove nginx`     | `dnf remove nginx`   |
| Upgrade all          | `apt upgrade`          | `dnf upgrade`        |
| Search               | `apt search nginx`     | `dnf search nginx`   |
| Info                 | `apt show nginx`       | `dnf info nginx`     |
| List installed       | `apt list --installed` | `dnf list installed` |
| Find file owner      | `dpkg -S /path`        | `dnf provides /path` |
| List package files   | `dpkg -L nginx`        | `rpm -ql nginx`      |

## Part 3: Repository Management (APT Sources)

### Source List Files

Location: `/etc/apt/sources.list` and `/etc/apt/sources.list.d/*.list`

**Example** **`/etc/apt/sources.list`** **(Ubuntu 22.04):**

```bash
# Main repository (officially supported)
deb http://archive.ubuntu.com/ubuntu jammy main restricted
deb http://archive.ubuntu.com/ubuntu jammy-updates main restricted
deb http://archive.ubuntu.com/ubuntu jammy-backports main restricted

# Universe (community-maintained)
deb http://archive.ubuntu.com/ubuntu jammy universe
deb http://archive.ubuntu.com/ubuntu jammy-updates universe

# Multiverse (non-free software)
deb http://archive.ubuntu.com/ubuntu jammy multiverse
deb http://archive.ubuntu.com/ubuntu jammy-updates multiverse

# Security updates
deb http://security.ubuntu.com/ubuntu jammy-security main restricted universe multiverse
```

### Source Line Format

```
deb [options] uri suite component1 component2 ...
deb-src [options] uri suite component1 component2 ...
```

| Field       | Meaning                    | Example                                                  |
| ----------- | -------------------------- | -------------------------------------------------------- |
| `deb`       | Binary packages            | `deb`                                                    |
| `deb-src`   | Source packages (optional) | `deb-src`                                                |
| `uri`       | Repository URL             | `http://archive.ubuntu.com/ubuntu`                       |
| `suite`     | Release codename           | `jammy` (22.04), `focal` (20.04), `bullseye` (Debian 11) |
| `component` | Package category           | `main`, `restricted`, `universe`, `multiverse`           |

**Ubuntu release codenames:**

| Version | Codename        | LTS?           |
| ------- | --------------- | -------------- |
| 24.04   | Noble Numbat    | Yes            |
| 22.04   | Jammy Jellyfish | Yes            |
| 20.04   | Focal Fossa     | Yes            |
| 18.04   | Bionic Beaver   | Yes (EOL 2023) |

**Debian release codenames:**

| Version | Codename |
| ------- | -------- |
| 12      | Bookworm |
| 11      | Bullseye |
| 10      | Buster   |

### Managing Repositories

```bash
# List all enabled repositories
grep -r "^deb " /etc/apt/sources.list /etc/apt/sources.list.d/

# Add a repository (PPA – Personal Package Archive)
sudo add-apt-repository ppa:nginx/stable
sudo apt update

# Remove a PPA
sudo add-apt-repository --remove ppa:nginx/stable

# Add repository manually (create .list file)
echo "deb https://nginx.org/packages/ubuntu/ jammy nginx" | sudo tee /etc/apt/sources.list.d/nginx.list
sudo apt update

# Add repository with key
curl -fsSL https://nginx.org/keys/nginx_signing.key | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/nginx.gpg
sudo apt update
```

### Repository Priority (APT Pinning)

Pin a package to a specific version or repository.

```bash
# Create pin file
sudo cat > /etc/apt/preferences.d/nginx << EOF
Package: nginx
Pin: version 1.18.*
Pin-Priority: 1001
EOF

# Priorities:
# 1001 = force this version
# 990 = prefer this repository
# 500 = default
# 100 = install only if no other version
# -1 = never install

# See pin priorities
apt-cache policy nginx
```

## Part 4: Working with DPKG Packages Directly

### Downloading Packages Without Installing

```bash
# Download to current directory
apt download nginx

# Download with dependencies (to create offline bundle)
apt-get download $(apt-cache depends --recurse --no-recommends --no-suggests --no-conflicts --no-breaks --no-replaces --no-enhances nginx | grep "^\w" | sort -u)
```

### Inspecting DEB Files

```bash
# Show package information from .deb file
dpkg -I package.deb

# List files in .deb (without installing)
dpkg -c package.deb

# Extract contents
dpkg -x package.deb /tmp/extract/
dpkg -e package.deb /tmp/extract/DEBIAN  # Extract control info
```

### Creating Local APT Repository

```bash
# 1. Install reprepro (repository tool)
sudo apt install reprepro

# 2. Create repository structure
mkdir -p /var/local/apt-repo/conf
cat > /var/local/apt-repo/conf/distributions << EOF
Codename: focal
Components: main
Architectures: amd64
EOF

# 3. Add packages
reprepro -b /var/local/apt-repo includedeb focal *.deb

# 4. Add to sources.list
echo "deb file:///var/local/apt-repo focal main" | sudo tee /etc/apt/sources.list.d/local.list
sudo apt update
```

## Part 5: Common Debian/Ubuntu Operations

### Installing from Ubuntu Repositories

```bash
# Standard install
sudo apt update
sudo apt install nginx

# Install build dependencies for a package
sudo apt build-dep nginx

# Install recommended packages (default: true)
sudo apt install --install-recommends nginx

# Install without recommended packages (minimal)
sudo apt install --no-install-recommends nginx
```

### Holding Packages (Prevent Upgrades)

```bash
# Hold a package at current version
sudo apt-mark hold nginx

# Unhold
sudo apt-mark unhold nginx

# Show held packages
apt-mark showhold

# Alternative: echo package hold
echo "nginx hold" | sudo dpkg --set-selections
dpkg --get-selections | grep hold
```

### Package Version Control

```bash
# Show available versions
apt list -a nginx

# Install specific version
sudo apt install nginx=1.18.0-0ubuntu1

# Upgrade to specific version (if newer)
sudo apt install nginx=1.20.0-0ubuntu1
```

### Fixing Broken Packages

```bash
# Attempt to fix broken dependencies
sudo apt --fix-broken install

# Reconfigure all partially configured packages
sudo dpkg --configure -a

# Force remove broken package
sudo dpkg --remove --force-remove-reinstreq package-name
```

## Part 6: Troubleshooting DPKG/APT

### Problem 1: "Could not get lock /var/lib/dpkg/lock"

```bash
# Another package manager is running
# Wait or kill stale process
sudo rm /var/lib/dpkg/lock-frontend
sudo rm /var/lib/dpkg/lock
sudo dpkg --configure -a
```

### Problem 2: "Unable to fetch" / "Hash Sum mismatch"

```bash
# Clear cache and retry
sudo apt clean
sudo apt update
```

### Problem 3: "Unmet dependencies" loop

```bash
# Force fix
sudo apt --fix-broken install

# If still failing, try:
sudo apt-get -f install
sudo dpkg --configure -a
```

### Problem 4: "dpkg: error processing package"

```bash
# Reinstall problematic package
sudo apt install --reinstall package-name

# Force remove if reinstall fails
sudo dpkg --remove --force-remove-reinstreq package-name
sudo apt autoremove
```

### Problem 5: GPG Key Error

```bash
# Error: "The following signatures couldn't be verified"
# Add missing key
sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys <KEY_ID>

# Modern method (Ubuntu 20.04+)
curl -fsSL https://example.com/key.gpg | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/example.gpg
```

## Part 7: APT vs DNF: When to Use Which

| Task                 | Debian/Ubuntu (APT)                       | RHEL/Rocky (DNF)                       |
| -------------------- | ----------------------------------------- | -------------------------------------- |
| Install a web server | `apt install nginx`                       | `dnf install nginx`                    |
| Search for package   | `apt search nginx`                        | `dnf search nginx`                     |
| Update all packages  | `apt upgrade`                             | `dnf upgrade`                          |
| Add repository       | `add-apt-repository` or edit sources.list | Create `.repo` file                    |
| Remove with config   | `apt purge nginx`                         | `dnf remove nginx` (config may remain) |
| Low-level query      | `dpkg -L nginx`                           | `rpm -ql nginx`                        |
| Package file owner   | `dpkg -S /etc/nginx.conf`                 | `dnf provides /etc/nginx.conf`         |

## Summary Table: DPKG vs APT

| Operation             | DPKG (Low-Level)      | APT (High-Level)                    |
| --------------------- | --------------------- | ----------------------------------- |
| Install local .deb    | `dpkg -i package.deb` | `apt install ./package.deb`         |
| Install from repo     | N/A                   | `apt install nginx`                 |
| Remove (keep config)  | `dpkg -r nginx`       | `apt remove nginx`                  |
| Purge (remove config) | `dpkg -P nginx`       | `apt purge nginx`                   |
| List installed        | `dpkg -l`             | `apt list --installed`              |
| Package info          | `dpkg -s nginx`       | `apt show nginx`                    |
| List package files    | `dpkg -L nginx`       | `dpkg -L nginx` (no apt equivalent) |
| Find file owner       | `dpkg -S /path`       | `dpkg -S /path`                     |
| Update package lists  | N/A                   | `apt update`                        |
| Upgrade all           | N/A                   | `apt upgrade`                       |
| Search                | N/A                   | `apt search`                        |

### APT Command Reference

| Command            | Purpose                                       |
| ------------------ | --------------------------------------------- |
| `apt update`       | Refresh package lists from repositories       |
| `apt upgrade`      | Upgrade all upgradable packages               |
| `apt full-upgrade` | Smart upgrade (handles dependencies)          |
| `apt install`      | Install package(s)                            |
| `apt remove`       | Remove package(s) (keep config)               |
| `apt purge`        | Remove package(s) (remove config)             |
| `apt autoremove`   | Remove unused dependencies                    |
| `apt search`       | Search for packages                           |
| `apt show`         | Show package details                          |
| `apt list`         | List packages (--installed, --upgradable, -a) |
| `apt policy`       | Show version and repository priority          |
| `apt edit-sources` | Edit sources.list safely                      |
| `apt clean`        | Remove all .deb files from cache              |
| `apt autoclean`    | Remove obsolete .deb files                    |

### Repository Components (Ubuntu)

| Component    | Description                                |
| ------------ | ------------------------------------------ |
| `main`       | Officially supported, open source          |
| `restricted` | Supported, not fully open source (drivers) |
| `universe`   | Community-maintained, open source          |
| `multiverse` | Not open source, not supported             |
