## Cheatsheet: Nginx Core Concepts

### Nginx Architecture

| Component | Role |
|-----------|------|
| **Master Process** | Reads config, binds ports, manages workers |
| **Worker Process** | Handles requests (event-driven, non-blocking) |
| **Cache Loader** | Loads disk cache into memory |
| **Cache Manager** | Manages cache expiration |

### Nginx Commands

| Command | Purpose |
|---------|---------|
| `sudo systemctl start nginx` | Start Nginx |
| `sudo systemctl stop nginx` | Stop Nginx |
| `sudo systemctl reload nginx` | Reload config (zero downtime) |
| `sudo systemctl restart nginx` | Restart (brief downtime) |
| `sudo nginx -t` | Test configuration syntax |
| `sudo nginx -T` | Show full configuration |
| `sudo nginx -s reload` | Graceful reload |
| `sudo nginx -s quit` | Graceful stop |

### Configuration Structure

```nginx
user www-data;
worker_processes auto;
pid /run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    
    server {
        listen 80;
        server_name example.com;
        root /var/www/example.com;
        
        location / {
            try_files $uri $uri/ =404;
        }
    }
}
```

### Static Serving Directives

| Directive | Purpose | Example |
|-----------|---------|---------|
| `root` | Document root (appends URI) | `root /var/www/html` |
| `alias` | Path replacement | `alias /var/www/static/` |
| `index` | Default files | `index index.html` |
| `try_files` | Sequential file check | `try_files $uri $uri/ =404` |
| `error_page` | Custom error pages | `error_page 404 /404.html` |
| `expires` | Cache control | `expires 30d` |
| `autoindex` | Directory listing | `autoindex on` |

### Location Matching Priority

| Priority | Type | Syntax |
|----------|------|--------|
| 1 | Exact match | `location = /path` |
| 2 | Preferential prefix | `location ^~ /path` |
| 3 | Regex (case-sensitive) | `location ~ \.php$` |
| 4 | Regex (case-insensitive) | `location ~* \.jpg$` |
| 5 | Prefix (longest match) | `location /path` |

### Location Matching Examples

| URL | Location Match | Why |
|-----|----------------|-----|
| `/` | `location = /` | Exact match |
| `/static/style.css` | `location ^~ /static/` | Preferential prefix |
| `/api/users` | `location /` | Prefix match |
| `/image.jpg` | `location ~ \.jpg$` | Regex match |

### Cache Headers

| Header | Meaning |
|--------|---------|
| `public` | Cacheable by any cache |
| `private` | Cacheable only by browser |
| `no-cache` | Revalidate before using |
| `no-store` | Never cache |
| `immutable` | File will not change |

## Comparison Tables

### root vs alias

| Aspect | root | alias |
|--------|------|-------|
| **Path construction** | root + URI | alias (replaces URI) |
| **Trailing slash requirement** | Flexible | Critical |
| **Use case** | Main document root | Serving from outside root |

### Nginx vs Apache

| Feature | Nginx | Apache |
|---------|-------|--------|
| **Architecture** | Event-driven, async | Process/thread per connection |
| **Static file performance** | Excellent | Good |
| **Memory usage (10k connections)** | ~5MB | ~20GB+ |
| **Configuration** | Declarative | Imperative (`.htaccess`) |
| **Dynamic content** | Proxy to backend | Modules (mod_php) |

### Location Types Comparison

| Type | Syntax | When to Use |
|------|--------|-------------|
| Exact | `location = /` | Homepage, exact paths |
| Preferential prefix | `location ^~ /static/` | Stop regex checking |
| Regex | `location ~ \.php$` | Dynamic handlers |
| Prefix | `location /` | Fallback for everything else |

## Interview Questions (Scenario-Based)

These questions assume only knowledge from Subchapter 7.1. Answers reference only concepts from 7.1.1 and 7.1.2.

### Question 1

**Scenario:** A developer serves a static website with Nginx. Images and CSS files load slowly. The HTML loads quickly.

**Question:** What configuration change would improve static asset loading time? Write the Nginx configuration.

**Answer:**

**Solution:** Add caching headers for static assets so browsers cache them locally.

```nginx
server {
    listen 80;
    server_name example.com;
    root /var/www/example.com;
    
    # HTML (no cache or short cache)
    location ~* \.html$ {
        expires -1;
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }
    
    # Static assets (long cache)
    location ~* \.(css|js|jpg|jpeg|png|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;  # Reduce log noise
    }
    
    location / {
        try_files $uri $uri/ =404;
    }
}
```

**Why this works:**
- Browsers cache CSS/JS/images locally after first load
- Subsequent page loads use cached assets (no network request)
- HTML reloads every time to get fresh content

**Verification:**
```bash
curl -I http://example.com/style.css
# Cache-Control: public, immutable
# Expires: Thu, 31 Dec 2037 23:59:59 GMT
```

### Question 2

**Scenario:** A site has URLs like `/about` that should serve `/about.html`, but without the `.html` extension in the URL.

**Question:** How would you configure Nginx to serve clean URLs? Write the configuration.

**Answer:**

**Solution:** Use `try_files` to check for `.html` extension.

```nginx
server {
    listen 80;
    server_name example.com;
    root /var/www/example.com;
    index index.html;
    
    location / {
        try_files $uri $uri.html $uri/ =404;
    }
}
```

**How it works:**
1. Request `/about`
2. Check if `/var/www/example.com/about` exists (file)
3. Check if `/var/www/example.com/about.html` exists (file)
4. Check if `/var/www/example.com/about/` exists (directory)
5. If none exist, return 404

**Alternative (more flexible):**
```nginx
location / {
    try_files $uri $uri/ @htmlext;
}

location @htmlext {
    rewrite ^/(.*)$ /$1.html break;
    try_files $uri =404;
}
```

### Question 3

**Scenario:** A user reports that `https://example.com/images/logo.png` returns 404, but the file exists at `/var/www/example.com/img/logo.png`.

**Question:** How would you fix this without moving the file? Write the Nginx configuration.

**Answer:**

**Solution:** Use `alias` to map the URL path to a different filesystem location.

```nginx
server {
    listen 80;
    server_name example.com;
    root /var/www/example.com;
    
    # Map /images/ to /var/www/example.com/img/
    location /images/ {
        alias /var/www/example.com/img/;
    }
    
    location / {
        try_files $uri $uri/ =404;
    }
}
```

**Important:** Ensure trailing slashes match:
```nginx
# Correct
location /images/ {
    alias /var/www/example.com/img/;
}

# Incorrect (missing trailing slash on alias)
location /images/ {
    alias /var/www/example.com/img;
}
# URL: /images/logo.png → /var/www/example.com/imglogo.png (wrong!)
```

### Question 4

**Scenario:** An Nginx server has multiple location blocks:
```nginx
location = / {
    root /var/www/home;
}
location ^~ /static/ {
    alias /var/www/static/;
}
location ~ \.jpg$ {
    root /var/www/images;
}
location / {
    root /var/www/default;
}
```

**Question:** For the URL `/static/logo.jpg`, which location block matches? Why?

**Answer:**

**Matched location:** `location ^~ /static/`

**Why:**
1. Nginx checks locations in priority order
2. `location = /` – no match (URL is `/static/logo.jpg`, not `/`)
3. `location ^~ /static/` – matches! (preferential prefix)
4. The `^~` modifier stops further regex checking
5. Regex `location ~ \.jpg$` is never evaluated

**Result:** File served from `/var/www/static/logo.jpg`

**What if `^~` wasn't there?**
- Without `^~`, Nginx would check regex matches
- `location ~ \.jpg$` would also match
- Nginx would use the first regex match (order matters)

### Question 5

**Scenario:** After deploying a new Nginx configuration, `sudo nginx -t` shows:
```
nginx: [emerg] "root" directive is duplicate in /etc/nginx/sites-enabled/example.com:12
```

**Question:** What does this error mean? How would you fix it?

**Answer:**

**Error meaning:** You have multiple `root` directives in the same `server` block or conflicting contexts. Nginx only allows one `root` per server/location (unless overridden in nested locations).

**Common causes:**

1. **Duplicate root in same block:**
```nginx
server {
    root /var/www/html;  # First root
    # ... some config
    root /var/www/example;  # Second root (ERROR!)
}
```

2. **Root in server and location without override intent:**
```nginx
server {
    root /var/www/default;
    location / {
        root /var/www/default;  # Duplicate (redundant, not needed)
    }
}
```

**Fixes:**

**Fix 1 – Remove duplicate:**
```nginx
server {
    root /var/www/example;  # Keep one
    # Remove the other root directive
}
```

**Fix 2 – Use location-specific root (override):**
```nginx
server {
    root /var/www/default;
    
    location /admin {
        root /var/www/admin;  # Override for /admin only (allowed)
    }
}
```

**Fix 3 – Use alias instead of root for different paths:**
```nginx
server {
    root /var/www/default;
    
    location /images {
        alias /var/www/images;  # Use alias instead of another root
    }
}
```

**Verification after fix:**
```bash
sudo nginx -t
# nginx: configuration file test is successful
sudo systemctl reload nginx
```
