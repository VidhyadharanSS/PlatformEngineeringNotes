## Cheatsheet: Error Handling and String Manipulation

### Exit Codes

| Code | Meaning                  | When to Use        |
| ---- | ------------------------ | ------------------ |
| 0    | Success                  | Normal exit        |
| 1    | General error            | Catch-all failure  |
| 2    | Misuse (wrong arguments) | Invalid input      |
| 126  | Command not executable   | Permission issue   |
| 127  | Command not found        | Missing dependency |

### Set Options (Production Header)

```bash
#!/usr/bin/env bash
set -euo pipefail
```

| Option            | Effect                              |
| ----------------- | ----------------------------------- |
| `set -e`          | Exit on any command failure         |
| `set -u`          | Error on unset variables            |
| `set -o pipefail` | Pipeline fails if any command fails |
| `set -x`          | Print commands (debugging)          |
| `set +x`          | Disable debugging                   |

### Trap Signals

| Signal | Number | Trigger        | Common Use                 |
| ------ | ------ | -------------- | -------------------------- |
| `EXIT` | 0      | Script ends    | Cleanup temp files         |
| `INT`  | 2      | Ctrl+C         | Graceful shutdown          |
| `TERM` | 15     | `kill` command | Cleanup before termination |

```bash
trap 'cleanup' EXIT INT TERM
```

### Parameter Expansion (String Manipulation)

| Operation             | Syntax                 | Example                |
| --------------------- | ---------------------- | ---------------------- |
| Length                | `${#var}`              | `len=${#str}`          |
| Remove prefix (short) | `${var#pattern}`       | `file=${path##*/}`     |
| Remove suffix (short) | `${var%pattern}`       | `base=${file%.*}`      |
| Replace first         | `${var/old/new}`       | `text=${text/foo/bar}` |
| Replace all           | `${var//old/new}`      | `text=${text// /_}`    |
| Uppercase all         | `${var^^}`             | `"hello"` → `"HELLO"`  |
| Lowercase all         | `${var,,}`             | `"HELLO"` → `"hello"`  |
| Substring             | `${var:offset:length}` | `sub=${str:2:3}`       |
| Default value         | `${var:-default}`      | `port="${PORT:-8080}"` |

### Case Conversion

| Syntax     | Effect          | Example               |
| ---------- | --------------- | --------------------- |
| `${var^}`  | Uppercase first | `"hello"` → `"Hello"` |
| `${var^^}` | Uppercase all   | `"hello"` → `"HELLO"` |
| `${var,}`  | Lowercase first | `"HELLO"` → `"hELLO"` |
| `${var,,}` | Lowercase all   | `"HELLO"` → `"hello"` |

### Regular Expressions with `=~`

| Pattern    | Meaning         | Example       |
| ---------- | --------------- | ------------- |
| `^`        | Start of string | `^start`      |
| `$`        | End of string   | `end$`        |
| `.`        | Any character   | `c.t`         |
| `[0-9]`    | Digit           | `[0-9]{3}`    |
| `[a-zA-Z]` | Letter          | `[A-Za-z]+`   |
| `+`        | One or more     | `a+`          |
| `*`        | Zero or more    | `a*`          |
| `?`        | Zero or one     | `colou?r`     |
| `{3}`      | Exactly 3       | `[0-9]{3}`    |
| `\|`       | OR              | `error\|fail` |
| `()`       | Capture group   | `(error)`     |

### BASH\_REMATCH

| Index                | Contains             |
| -------------------- | -------------------- |
| `${BASH_REMATCH[0]}` | Entire match         |
| `${BASH_REMATCH[1]}` | First capture group  |
| `${BASH_REMATCH[2]}` | Second capture group |

### Logging Functions Template

```bash
log_info() {
    echo "[INFO] $(date '+%Y-%m-%d %H:%M:%S') - $*"
}

log_error() {
    echo "[ERROR] $(date '+%Y-%m-%d %H:%M:%S') - $*" >&2

```

### Debugging Techniques

| Method               | Command                     |
| -------------------- | --------------------------- |
| Syntax check         | `bash -n script.sh`         |
| Trace execution      | `bash -x script.sh`         |
| Debug with env var   | `DEBUG=1 ./script.sh`       |
| Line-by-line tracing | `trap 'echo $LINENO' DEBUG` |

## Comparison Tables

### `set -e` vs Manual Error Checking

| Approach           | Pros                       | Cons                                 |
| ------------------ | -------------------------- | ------------------------------------ |
| `set -e`           | Simple, catches all errors | Can't recover from expected failures |
| Manual `if` checks | Fine-grained control       | Verbose, easy to miss checks         |
| Combined           | Both safety and control    | Most robust                          |

### Parameter Expansion vs sed/awk

| Operation     | Parameter Expansion  | sed/awk                        |
| ------------- | -------------------- | ------------------------------ |
| Remove prefix | `${var#prefix}`      | `sed 's/^prefix//'`            |
| Replace all   | `${var//old/new}`    | `sed 's/old/new/g'`            |
| Substring     | `${var:2:3}`         | `awk '{print substr($0,3,3)}'` |
| Performance   | Very fast (built-in) | Slower (external process)      |
| Complexity    | Simple patterns      | Complex regex                  |

### Globbing vs Regex

| Feature        | Globbing (`[[ == ]]`) | Regex (`[[ =~ ]]`)                   |
| -------------- | --------------------- | ------------------------------------ |
| `*`            | Any string            | Zero or more of previous (need `.*`) |
| `?`            | Single character      | Zero or one of previous              |
| `+`            | Not available         | One or more of previous              |
| Capture groups | No                    | Yes (`BASH_REMATCH`)                 |
| Use case       | Filename matching     | Pattern extraction                   |

### Question 1

**Scenario:** A deployment script runs without `set -e`. A command fails early, but the script continues and deletes critical files before the error is noticed.

**Question:** What is the risk of running without `set -e`? Show how to fix the script. Also explain how to handle cases where a command is allowed to fail.

**Answer:**

**Risk:** Without `set -e`, the script continues executing after any command failure. This can lead to:

* Data corruption (deleting files when a prerequisite command failed)

* Inconsistent state (partial configuration changes)

* Silent failures (no one notices the error until it's too late)

**Example of dangerous script without** **`set -e`:**

```bash
#!/usr/bin/env bash
# DANGEROUS – no error handling

cd /var/www/myapp
rm -rf *  # If cd failed, this runs in current directory!
```

**Fix with** **`set -e`:**

```bash
#!/usr/bin/env bash
set -euo pipefail

cd /var/www/myapp  # Script exits if this fails
rm -rf *           # Never reaches here if cd failed
```

**Handling expected failures (commands that may legitimately fail):**

```bash
#!/usr/bin/env bash
set -euo pipefail

# Method 1: Use `|| true` to ignore failure
grep "pattern" file.txt || true

# Method 2: Use `if` to handle both cases
if grep "pattern" file.txt; then
    echo "Found pattern"
else
    echo "Pattern not found (this is OK)"
fi

# Method 3: Temporarily disable `set -e`
set +e
grep "pattern" file.txt
result=$?
set -e

if [[ $result -eq 0 ]]; then
    echo "Found"
fi

# Method 4: Use `||` with a no-op command
grep "pattern" file.txt || :

# Method 5: Command substitution (doesn't trigger `set -e`)
matches=$(grep "pattern" file.txt || true)
```

**Best practice for commands that may fail legitimately:**

```bash
#!/usr/bin/env bash
set -euo pipefail

# Function that handles expected failures
remove_if_exists() {
    local file="$1"
    if [[ -f "$file" ]]; then
        rm "$file"
    fi
    # Return success always (0)
    return 0
}

remove_if_exists "/tmp/tempfile.txt"
```

### Question 2

**Scenario:** A script creates temporary files but does not clean them up on exit. When the script is interrupted by Ctrl+C or fails, temp files accumulate.

**Question:** How would you ensure temporary files are always deleted? Write a script that demonstrates proper cleanup using `trap`.

**Answer:**

**Solution:** Use `trap` with `EXIT` signal to run cleanup regardless of how the script exits (normal, error, or interrupt).

```bash
#!/usr/bin/env bash
set -euo pipefail

# Create temporary files safely
temp_config=$(mktemp)
temp_log=$(mktemp)
temp_data=$(mktemp -d)  # Directory

# Cleanup function
cleanup() {
    local exit_code=$?
    echo "Cleaning up temporary files..."
    
    # Remove individual files
    rm -f "$temp_config" "$temp_log"
    
    # Remove directory recursively
    rm -rf "$temp_data"
    
    if [[ $exit_code -eq 0 ]]; then
        echo "Script completed successfully"
    else
        echo "Script exited with error code $exit_code"
    fi
}

# Set trap for multiple signals
trap cleanup EXIT INT TERM

# Rest of script
echo "Temp config: $temp_config"
echo "Temp log: $temp_log"
echo "Temp dir: $temp_data"

# Write to temp files
echo "config_data" > "$temp_config"
echo "log_data" > "$temp_log"

# Simulate work
sleep 2

# Script exits normally (cleanup runs)
exit 0
```

**Testing the trap:**

```bash
# Test with Ctrl+C
./script.sh
# Press Ctrl+C during sleep – cleanup still runs

# Test with error
./script.sh
# Script has error – cleanup still runs

# Test with kill
./script.sh &
kill %1  # Sends TERM – cleanup runs
```

**Creating a safe temporary file library:**

```bash
#!/usr/bin/env bash

# Global array to track temp files
declare -a TEMP_FILES=()

# Create temp file and register for cleanup
create_temp_file() {
    local file
    file=$(mktemp)
    TEMP_FILES+=("$file")
    echo "$file"
}

# Create temp directory
create_temp_dir() {
    local dir
    dir=$(mktemp -d)
    TEMP_FILES+=("$dir")
    echo "$dir"
}

# Cleanup all registered temp files
cleanup_temp_files() {
    for item in "${TEMP_FILES[@]}"; do
        if [[ -f "$item" ]]; then
            rm -f "$item"
        elif [[ -d "$item" ]]; then
            rm -rf "$item"
        fi
    done
}

# Set trap automatically
trap cleanup_temp_files EXIT INT TERM

# Usage
config=$(create_temp_file)
data_dir=$(create_temp_dir)

echo "Config: $config"
echo "Data dir: $data_dir"
```

### Question 3

**Scenario:** A script needs to parse a log file with lines like:

```
2024-01-16 10:30:45,123 ERROR [auth] Failed login for user 'alice' from 192.168.1.100
2024-01-16 10:31:20,456 WARN  [db] Connection pool reaching limit
2024-01-16 10:32:05,789 INFO  [api] Request processed in 234ms
```

**Question:** Write a script that:

* Extracts the timestamp, level, component, and message

* Counts errors per component

* Outputs only ERROR lines with IP addresses

**Answer:**

```bash
#!/usr/bin/env bash
set -euo pipefail

LOG_FILE="${1:-/var/log/myapp.log}"
declare -A error_count

# Pattern with capture groups
pattern='([0-9]{4}-[0-9]{2}-[0-9]{2} [0-9]{2}:[0-9]{2}:[0-9]{2},[0-9]{3}) (ERROR|WARN|INFO) \[([^\]]+)\] (.*)'

process_log() {
    local line_num=0
    
    while IFS= read -r line; do
        line_num=$((line_num + 1))
        
        if [[ "$line" =~ $pattern ]]; then
            timestamp="${BASH_REMATCH[1]}"
            level="${BASH_REMATCH[2]}"
            component="${BASH_REMATCH[3]}"
            message="${BASH_REMATCH[4]}"
            
            # Count errors per component
            if [[ "$level" == "ERROR" ]]; then
                error_count["$component"]=$((error_count["$component"] + 1))
                
                # Extract IP from error message
                if [[ "$message" =~ [0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3} ]]; then
                    ip="${BASH_REMATCH[0]}"
                    echo "[ERROR] $timestamp [$component] IP: $ip - $message"
                else
                    echo "[ERROR] $timestamp [$component] $message"
                fi
            elif [[ "$level" == "WARN" ]]; then
                echo "[WARN]  $timestamp [$component] $message"
            fi
        else
            echo "WARNING: Could not parse line $line_num: $line"
        fi
    done < "$LOG_FILE"
    
    echo "=========================================="
    echo "Error count by component:"
    for component in "${!error_count[@]}"; do
        echo "  $component: ${error_count[$component]}"
    done
}

# Check if log file exists
if [[ ! -f "$LOG_FILE" ]]; then
    echo "ERROR: Log file not found: $LOG_FILE"
    exit 1
fi

process_log
```

### Question 4

**Scenario:** A configuration management script needs to update a value in a properties file. The file has lines like:

```
# This is a comment
db.host=localhost
db.port=5432
db.user=appuser
# db.password=oldpassword
```

**Question:** Write a function that:

* Updates `db.host` to a new value (if exists, otherwise appends)

* Comments out the old `db.password` line and adds a new one

* Preserves comments and formatting

**Answer:**

```bash
#!/usr/bin/env bash
set -euo pipefail

CONFIG_FILE="/etc/myapp/config.properties"

# Function to update property
update_property() {
    local key="$1"
    local value="$2"
    local file="$3"
    
    # Escape special characters for sed
    local escaped_key=$(echo "$key" | sed 's/[.[\*^$]/\\&/g')
    
    if grep -q "^[[:space:]]*${escaped_key}[[:space:]]*=" "$file"; then
        # Update existing property
        sed -i "s/^[[:space:]]*${escaped_key}[[:space:]]*=.*/${key}=${value}/" "$file"
        echo "Updated: $key=$value"
    else
        # Append new property
        echo "${key}=${value}" >> "$file"
        echo "Added: $key=$value"
    fi
}

# Function to comment out property
comment_property() {
    local key="$1"
    local file="$2"
    
    local escaped_key=$(echo "$key" | sed 's/[.[\*^$]/\\&/g')
    
    if grep -q "^[[:space:]]*${escaped_key}[[:space:]]*=" "$file"; then
        sed -i "s/^[[:space:]]*${escaped_key}[[:space:]]*=/# &/" "$file"
        echo "Commented out: $key"
    fi
}

# Function to add new password (comment old, add new)
rotate_password() {
    local new_password="$1"
    local file="$2"
    
    # Comment out existing password line (if any)
    comment_property "db.password" "$file"
    
    # Add new password at the end
    echo "db.password=${new_password}" >> "$file"
    echo "Added new db.password"
}

# Backup before changes
backup_file() {
    local file="$1"
    local backup="${file}.$(date +%Y%m%d_%H%M%S).bak"
    cp "$file" "$backup"
    echo "Backup created: $backup"
}

# Main update process
main() {
    local new_host="${1:-db.example.com}"
    local new_password="${2:-SecurePass123}"
    
    if [[ ! -f "$CONFIG_FILE" ]]; then
        echo "ERROR: Config file not found: $CONFIG_FILE"
        exit 1
    fi
    
    # Backup before changes
    backup_file "$CONFIG_FILE"
    
    # Update properties
    update_property "db.host" "$new_host" "$CONFIG_FILE"
    update_property "db.port" "5432" "$CONFIG_FILE"
    rotate_password "$new_password" "$CONFIG_FILE"
    
    echo "Configuration updated successfully"
}

main "$@"
```

### Question 5

**Scenario:** A backup script must validate that a directory path is safe before deleting old backups. The validation rules:

* Path must be absolute (start with `/`)

* Path must be under `/backup` or `/data/backup`

* Path must not be `/` or `/backup` itself (must have at least one subdirectory)

**Question:** Write a validation function using regex and string manipulation. Include error messages for each failure case.

**Answer:**

```bash
#!/usr/bin/env bash
set -euo pipefail

validate_backup_path() {
    local path="$1"
    
    # Rule 1: Must be absolute path (start with /)
    if [[ ! "$path" =~ ^/ ]]; then
        echo "ERROR: Path must be absolute (start with /): $path"
        return 1
    fi
    
    # Rule 2: Must be under /backup or /data/backup
    if [[ ! "$path" =~ ^/(backup|data/backup)/ ]]; then
        echo "ERROR: Path must be under /backup or /data/backup: $path"
        return 1
    fi
    
    # Rule 3: Must not be the root backup directory itself
    if [[ "$path" == "/backup" ]] || [[ "$path" == "/data/backup" ]]; then
        echo "ERROR: Cannot operate on root backup directory: $path"
        return 1
    fi
    
    # Rule 4: Must have at least one subdirectory after the base
    local subpath="${path#/backup/}"
    subpath="${subpath#/data/backup/}"
    
    if [[ -z "$subpath" ]]; then
        echo "ERROR: Path must include at least one subdirectory: $path"
        return 1
    fi
    
    # Rule 5: Path must exist and be a directory
    if [[ ! -d "$path" ]]; then
        echo "ERROR: Directory does not exist: $path"
        return 1
    fi
    
    # Rule 6: Path must not contain any symlinks (security)
    if [[ -L "$path" ]]; then
        echo "ERROR: Path contains symlink (unsafe): $path"
        return 1
    fi
    
    # Rule 7: Path must not contain ".." (parent directory traversal)
    if [[ "$path" =~ \.\. ]]; then
        echo "ERROR: Path contains '..' (unsafe): $path"
        return 1
    fi
    
    echo "SUCCESS: Path validation passed: $path"
    return 0
}

# Function to clean old backups (safe version)
clean_old_backups() {
    local backup_dir="$1"
    local days="${2:-30}"
    
    if ! validate_backup_path "$backup_dir"; then
        echo "Aborting cleanup due to validation failure"
        return 1
    fi
    
    echo "Cleaning backups older than $days days in $backup_dir"
    
    # Use find with validated path (safe)
    find "$backup_dir" -type f -name "*.tar.gz" -mtime "+$days" -delete
    find "$backup_dir" -type d -empty -delete 2>/dev/null || true
    
    echo "Cleanup completed"
}

# Test cases
test_validation() {
    echo "=== Testing validation ==="
    
    # Valid paths
    validate_backup_path "/backup/myapp" && echo "✓ Valid"
    validate_backup_path "/data/backup/database" && echo "✓ Valid"
    
    # Invalid paths
    validate_backup_path "relative/path" || echo "✗ Rejected (relative)"
    validate_backup_path "/tmp/backup" || echo "✗ Rejected (wrong base)"
    validate_backup_path "/backup" || echo "✗ Rejected (root backup)"
    validate_backup_path "/backup/" || echo "✗ Rejected (trailing slash)"
    validate_backup_path "/backup/../etc" || echo "✗ Rejected (parent traversal)"
}

# Run tests
test_validation

# Example usage
if validate_backup_path "/backup/myapp-20240116"; then
    clean_old_backups "/backup/myapp-20240116" 7
fi
```

**Enhanced version with whitelist for extra safety:**

```bash
#!/usr/bin/env bash
set -euo pipefail

# Whitelist of allowed backup directories
declare -a ALLOWED_BACKUP_DIRS=(
    "/backup/myapp"
    "/backup/database"
    "/data/backup/web"
    "/data/backup/logs"
)

is_allowed_directory() {
    local path="$1"
    for allowed in "${ALLOWED_BACKUP_DIRS[@]}"; do
        if [[ "$path" == "$allowed" || "$path" == "$allowed"/* ]]; then
            return 0
        fi
    done
    return 1
}

validate_backup_path_whitelist() {
    local path="$1"
    
    # First, run all basic validations
    if ! validate_backup_path "$path" 2>/dev/null; then
        return 1
    fi
    
    # Then check whitelist
    if ! is_allowed_directory "$path"; then
        echo "ERROR: Path not in whitelist: $path"
        echo "Allowed directories:"
        for allowed in "${ALLOWED_BACKUP_DIRS[@]}"; do
            echo "  - $allowed"
        done
        return 1
    fi
    
    echo "SUCCESS: Path validation passed (whitelist + checks)"
    return 0
}
```
