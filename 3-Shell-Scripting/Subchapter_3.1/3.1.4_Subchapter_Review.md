## Cheatsheet: Scripting Fundamentals

### Shebang (First Line)

| Shebang               | Interpreter    | Use Case         |
| --------------------- | -------------- | ---------------- |
| `#!/bin/bash`         | Bash           | Most scripts     |
| `#!/bin/sh`           | POSIX shell    | Portable scripts |
| `#!/usr/bin/env bash` | Bash from PATH | Most portable    |

### Variable Syntax

| Operation            | Syntax             | Example                           |
| -------------------- | ------------------ | --------------------------------- |
| Assignment           | `var=value`        | `name="Alice"`                    |
| Expansion            | `$var` or `${var}` | `echo "$name"`                    |
| Command substitution | `$(command)`       | `date=$(date)`                    |
| Default value        | `${var:-default}`  | `port="${PORT:-8080}"`            |
| Assign default       | `${var:=default}`  | `: "${LOG_LEVEL:=info}"`          |
| Error if unset       | `${var:?message}`  | `: "${DB_HOST:?Missing DB_HOST}"` |

### Special Variables

| Variable        | Meaning                  |
| --------------- | ------------------------ |
| `$0`            | Script name              |
| `$1`, `$2`, ... | Positional parameters    |
| `$#`            | Number of arguments      |
| `$@`            | All arguments (separate) |
| `$*`            | All arguments (single)   |
| `$?`            | Last command exit code   |
| `$$`            | Current PID              |
| `$!`            | Last background PID      |

### Quoting

| Quote            | Behavior                                                               |
| ---------------- | ---------------------------------------------------------------------- |
| `"double"`       | Allows variable expansion (`$var`) and command substitution (`$(cmd)`) |
| `'single'`       | Literal text (no expansion)                                            |
| `` `backtick` `` | Legacy command substitution (avoid)                                    |

### Arrays

| Operation          | Syntax           | Example                     |
| ------------------ | ---------------- | --------------------------- |
| Indexed array      | `arr=("a" "b")`  | `fruits=("apple" "banana")` |
| Access element     | `${arr[N]}`      | `echo "${fruits[0]}"`       |
| All elements       | `${arr[@]}`      | `for f in "${fruits[@]}"`   |
| Length             | `${#arr[@]}`     | `echo "${#fruits[@]}"`      |
| Associative array  | `declare -A arr` | `declare -A users`          |
| Access associative | `${arr[key]}`    | `echo "${users["alice"]}"`  |

### Subshells

| Syntax       | Purpose                                     |
| ------------ | ------------------------------------------- |
| `(command)`  | Run in child process                        |
| `$(command)` | Capture output                              |
| `<(command)` | Process substitution (treat output as file) |

### Conditionals (Test Operators)

| Category    | Operators                                      | <br /> | <br />  |
| ----------- | ---------------------------------------------- | :----- | :------ |
| **String**  | `=`, `==`, `!=`, `-z`, `-n`                    | <br /> | <br />  |
| **Numeric** | `-eq`, `-ne`, `-lt`, `-le`, `-gt`, `-ge`       | <br /> | <br />  |
| **File**    | `-e`, `-f`, `-d`, `-s`, `-r`, `-w`, `-x`, `-L` | <br /> | <br />  |
| **Logical** | `&&`, \`                                       | <br /> | `, `!\` |

### If Statement Syntax

```bash
if [[ condition ]]; then
    commands
elif [[ condition2 ]]; then
    commands
else
    commands
fi
```

### Case Statement Syntax

```bash
case "$variable" in
    pattern1)
        commands
        ;;
    pattern2|pattern3)
        commands
        ;;
    *)
        default_commands
        ;;
esac
```

### Loops

| Loop            | Syntax                                      |
| --------------- | ------------------------------------------- |
| `for` (list)    | `for item in list; do commands; done`       |
| `for` (C-style) | `for ((i=0; i<10; i++)); do commands; done` |
| `while`         | `while [[ condition ]]; do commands; done`  |
| `until`         | `until [[ condition ]]; do commands; done`  |
| `break`         | Exit loop                                   |
| `continue`      | Skip to next iteration                      |

### Functions

| Component      | Syntax                           |
| -------------- | -------------------------------- |
| Define         | `name() { commands; }`           |
| Call           | `name arg1 arg2`                 |
| Local variable | `local var=value`                |
| Return numeric | `return N` (0-255)               |
| Return output  | `echo "value"`                   |
| Access params  | `$1`, `$2`, `$@` inside function |

## Comparison Tables

### `[ ]` vs `[[ ]]`

| Feature           | `[ ]` (POSIX)                   | `[[ ]]` (Bash)      | <br /> | <br /> |
| ----------------- | ------------------------------- | ------------------- | :----- | :----- |
| Word splitting    | Yes (requires quoting)          | No                  | <br /> | <br /> |
| Pattern matching  | No                              | Yes (`==` with `*`) | <br /> | <br /> |
| Regex matching    | No                              | Yes (`=~`)          | <br /> | <br /> |
| Logical operators | `-a`, `-o`                      | `&&`, \`            | <br /> | \`     |
| `<`, `>`          | Lexicographic (requires escape) | Lexicographic       | <br /> | <br /> |
| Recommendation    | For POSIX compatibility         | For Bash scripts    | <br /> | <br /> |

### `$@` vs `$*`

| Variable | Behavior                              | When to Use                   |
| -------- | ------------------------------------- | ----------------------------- |
| `$@`     | Each argument as separate word        | Most cases (preserves spaces) |
| `$*`     | All arguments as single word          | Rare                          |
| `"$@"`   | Each argument as separate quoted word | **Recommended**               |
| `"$*"`   | All arguments as single quoted string | Joining arguments             |

### `break` vs `continue`

| Command      | Effect                               |
| ------------ | ------------------------------------ |
| `break`      | Exit loop completely                 |
| `break 2`    | Exit 2 nested loops                  |
| `continue`   | Skip to next iteration               |
| `continue 2` | Skip to next iteration of outer loop |

## Interview Questions (Scenario-Based)

These questions assume only knowledge from Subchapter 3.1. Answers reference only concepts from 3.1.1 and 3.1.2.

### Question 1

**Scenario:** A junior engineer writes a script with the following line:

```bash
files=$(ls *.txt)
for file in $files; do
    rm $file
done
```

**Question:** What are the potential bugs in this script? How would you rewrite it safely?

**Answer:**

**Potential bugs:**

1. **Word splitting:** If any filename contains spaces, `$files` will split into multiple words. Example: `"My Document.txt"` becomes two words: `"My"` and `"Document.txt"`.

2. **Globbing:** If a filename contains wildcard characters (`*`, `?`, `[`), they may be expanded again.

3. **Unquoted variable in** **`rm`:** `rm $file` will also split on spaces.

4. **Parsing** **`ls`** **output:** Parsing `ls` is generally discouraged because output is not safe for filenames with special characters.

**Safe rewrite using glob directly:**

```bash
#!/usr/bin/env bash

# Safe: Use glob directly, quoted variable
for file in *.txt; do
    # Check if file exists (in case no .txt files)
    if [[ -f "$file" ]]; then
        rm "$file"
    fi
done
```

**Even safer with** **`nullglob`** **(prevents literal** **`*.txt`** **when no matches):**

```bash
#!/usr/bin/env bash
shopt -s nullglob

for file in *.txt; do
    rm "$file"
done
```

**Alternative using array:**

```bash
#!/usr/bin/env bash
files=(*.txt)
for file in "${files[@]}"; do
    if [[ -f "$file" ]]; then
        rm "$file"
    fi
done
```

**Why the original fails with spaces:**

* Assume files: `"a.txt"`, `"b c.txt"`

* `files=$(ls *.txt)` → `files="a.txt b c.txt"`

* `for file in $files` (unquoted) → `file` becomes: `a.txt`, `b`, `c.txt`

* `rm $file` (unquoted) → tries to remove `a.txt`, `b`, `c.txt` (fails for `b`)

### Question 2

**Scenario:** A deployment script needs to check if a service is healthy before proceeding. The health endpoint is `http://localhost:8080/health` and should return `200 OK` with body `"healthy"`.

**Question:** Write a function `wait_for_health` that:

* Checks the endpoint every 2 seconds

* Times out after 30 seconds

* Returns 0 if healthy, 1 if timeout

* Uses appropriate conditionals and loops

**Answer:**

```bash
#!/usr/bin/env bash

wait_for_health() {
    local url="${1:-http://localhost:8080/health}"
    local timeout="${2:-30}"
    local interval="${3:-2}"
    local elapsed=0
    
    echo "Waiting for health endpoint: $url"
    
    while [[ $elapsed -lt $timeout ]]; do
        # Perform health check
        response=$(curl -s -o /dev/null -w "%{http_code}" "$url" 2>/dev/null)
        body=$(curl -s "$url" 2>/dev/null)
        
        if [[ "$response" -eq 200 ]] && [[ "$body" == "healthy" ]]; then
            echo "Service is healthy!"
            return 0
        fi
        
        echo "Still waiting... (${elapsed}s / ${timeout}s)"
        sleep "$interval"
        elapsed=$((elapsed + interval))
    done
    
    echo "ERROR: Timeout waiting for service to become healthy"
    return 1
}

# Usage
if wait_for_health "http://localhost:8080/health" 30 2; then
    echo "Proceeding with deployment..."
    # Deploy here
else
    echo "Deployment aborted"
    exit 1
fi
```

**Enhanced version with exponential backoff:**

```bash
wait_for_health_backoff() {
    local url="${1:-http://localhost:8080/health}"
    local max_wait="${2:-30}"
    local attempt=1
    local wait_time=1
    local elapsed=0
    
    while [[ $elapsed -lt $max_wait ]]; do
        response=$(curl -s -o /dev/null -w "%{http_code}" "$url" 2>/dev/null)
        body=$(curl -s "$url" 2>/dev/null)
        
        if [[ "$response" -eq 200 ]] && [[ "$body" == "healthy" ]]; then
            echo "Service healthy after ${elapsed}s (attempt $attempt)"
            return 0
        fi
        
        echo "Attempt $attempt failed (HTTP $response). Waiting ${wait_time}s..."
        sleep "$wait_time"
        elapsed=$((elapsed + wait_time))
        wait_time=$((wait_time * 2))  # Double backoff
        if [[ $wait_time -gt 10 ]]; then
            wait_time=10  # Cap at 10 seconds
        fi
        attempt=$((attempt + 1))
    done
    
    echo "ERROR: Service not healthy after ${max_wait}s"
    return 1
}
```

### Question 3

**Scenario:** You need to write a script that processes a CSV file with the following format:

```
name,age,city
Alice,30,New York
Bob,25,Los Angeles
Charlie,35,Chicago
```

**Question:** Write a script that:

* Reads the CSV file line by line (skipping header)

* For each person, prints "NAME (age) lives in CITY"

* Counts how many people are over 30

* Uses proper quoting and IFS

**Answer:**

```bash
#!/usr/bin/env bash

process_csv() {
    local csv_file="$1"
    local count_over_30=0
    local line_num=0
    
    if [[ ! -f "$csv_file" ]]; then
        echo "ERROR: File $csv_file not found"
        return 1
    fi
    
    # Read line by line with IFS to preserve spaces
    while IFS=',' read -r name age city; do
        line_num=$((line_num + 1))
        
        # Skip header
        if [[ $line_num -eq 1 ]]; then
            continue
        fi
        
        # Trim whitespace (optional)
        name=$(echo "$name" | xargs)
        age=$(echo "$age" | xargs)
        city=$(echo "$city" | xargs)
        
        # Print formatted output
        echo "$name ($age) lives in $city"
        
        # Count people over 30
        if [[ $age -gt 30 ]]; then
            count_over_30=$((count_over_30 + 1))
        fi
    done < "$csv_file"
    
    echo "--------------------------------"
    echo "Total people over 30: $count_over_30"
}

# Alternative using associative array for more complex processing
process_csv_advanced() {
    local csv_file="$1"
    declare -A age_sum_by_city
    declare -A count_by_city
    
    # Read header first
    IFS=',' read -r header < "$csv_file"
    
    while IFS=',' read -r name age city; do
        # Trim whitespace
        name="${name// /}"
        city="${city// /}"
        
        age_sum_by_city[$city]=$(( ${age_sum_by_city[$city]:-0} + age ))
        count_by_city[$city]=$(( ${count_by_city[$city]:-0} + 1 ))
    done < <(tail -n +2 "$csv_file")
    
    echo "Average age by city:"
    for city in "${!age_sum_by_city[@]}"; do
        avg=$(( age_sum_by_city[$city] / count_by_city[$city] ))
        echo "  $city: $avg years"
    done
}

# Run the script
process_csv "data.csv"
```

**Explanation of key points:**

* `IFS=','` sets field separator to comma (preserves spaces inside fields)

* `read -r` prevents backslash interpretation

* `tail -n +2` skips the header line

* `xargs` trims whitespace (simpler than parameter expansion for this case)

* Process substitution `< <(command)` avoids subshell variable scope issues

### Question 4

**Scenario:** A script uses a global variable `LOG_FILE` that is modified inside a function. Another function later reads the same variable expecting the original value, causing bugs.

**Question:** What is the problem? How would you fix it using proper variable scoping? Provide before and after code examples.

**Answer:**

**Problem:** The script uses global variables where local variables should be used, causing unintended side effects between functions.

**Buggy code (before):**

```bash
#!/usr/bin/env bash

LOG_FILE="/var/log/myapp.log"

setup_logging() {
    # Oops! This modifies the global LOG_FILE
    LOG_FILE="/tmp/setup.log"
    echo "Setup logging to $LOG_FILE"
}

process_data() {
    # Expects LOG_FILE to be /var/log/myapp.log, but it was changed!
    echo "Processing data..." >> "$LOG_FILE"
}

setup_logging
process_data  # Writes to /tmp/setup.log instead of /var/log/myapp.log!
```

**Fixed code (after):**

```bash
#!/usr/bin/env bash

# Global (read-only after setup)
readonly LOG_FILE="/var/log/myapp.log"

setup_logging() {
    # Use local variable – does not affect global
    local temp_log="/tmp/setup.log"
    echo "Setup logging to $temp_log"
    # Write to temp log if needed
    echo "Setup completed" >> "$temp_log"
}

process_data() {
    # Uses the global constant
    echo "Processing data..." >> "$LOG_FILE"
}

# Or use function parameters instead of globals
process_data_with_param() {
    local log_file="$1"
    echo "Processing data..." >> "$log_file"
}

setup_logging
process_data
process_data_with_param "$LOG_FILE"
```

**Best practices to avoid scoping issues:**

1. **Use** **`local`** **inside functions:**

```bash
my_function() {
    local temp_var="value"  # Not visible outside
}
```

1. **Use** **`readonly`** **for global constants:**

```bash
readonly CONFIG_DIR="/etc/myapp"
readonly MAX_RETRIES=5
```

1. **Pass values as parameters instead of using globals:**

```bash
# Instead of global LOG_FILE
log_message() {
    local log_file="$1"
    local message="$2"
    echo "$message" >> "$log_file"
}
```

1. **Use naming conventions to distinguish globals (e.g., uppercase):**

```bash
GLOBAL_CONFIG_FILE="/etc/myapp/config.conf"
global_counter=0  # Not recommended; use uppercase for globals

process_data() {
    local local_counter=$global_counter  # Intentional use
    local_counter=$((local_counter + 1))
    # ... do work
    global_counter=$local_counter  # Explicit update
}
```

1. **Use** **`declare -g`** **for intentional global variables (Bash 4.2+):**

```bash
set_global() {
    declare -g GLOBAL_VAR="explicitly global"
}
```

### Question 5

**Scenario:** You need to write a script that manages a whitelist of allowed IP addresses. The script should:

* Read IPs from a file `whitelist.txt` (one per line)

* Add a new IP (if not already present)

* Remove an IP (if present)

* List all IPs

**Question:** Implement the script using functions for each operation. Use arrays and proper argument handling with `case` statements.

**Answer:**

```bash
#!/usr/bin/env bash

# Configuration
readonly WHITELIST_FILE="${WHITELIST_FILE:-/etc/myapp/whitelist.txt}"
readonly TEMP_FILE="/tmp/whitelist.tmp"

# Function to load whitelist into array
load_whitelist() {
    local -n whitelist_ref="$1"  # Reference to array
    whitelist_ref=()
    
    if [[ -f "$WHITELIST_FILE" ]]; then
        while IFS= read -r ip; do
            # Skip empty lines and comments
            if [[ -n "$ip" && ! "$ip" =~ ^[[:space:]]*# ]]; then
                whitelist_ref+=("$ip")
            fi
        done < "$WHITELIST_FILE"
    fi
}

# Function to save array to file
save_whitelist() {
    local -n whitelist_ref="$1"
    
    # Create directory if it doesn't exist
    mkdir -p "$(dirname "$WHITELIST_FILE")"
    
    # Write to temp file then move (atomic)
    > "$TEMP_FILE"
    for ip in "${whitelist_ref[@]}"; do
        echo "$ip" >> "$TEMP_FILE"
    done
    mv "$TEMP_FILE" "$WHITELIST_FILE"
    
    echo "Whitelist saved to $WHITELIST_FILE"
}

# Function to check if IP is valid
is_valid_ip() {
    local ip="$1"
    if [[ $ip =~ ^[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}$ ]]; then
        IFS='.' read -r o1 o2 o3 o4 <<< "$ip"
        if [[ $o1 -le 255 && $o2 -le 255 && $o3 -le 255 && $o4 -le 255 ]]; then
            return 0
        fi
    fi
    return 1
}

# Function to add IP
add_ip() {
    local ip="$1"
    
    if ! is_valid_ip "$ip"; then
        echo "ERROR: Invalid IP address: $ip"
        return 1
    fi
    
    local -a whitelist
    load_whitelist whitelist
    
    # Check if already exists
    for existing_ip in "${whitelist[@]}"; do
        if [[ "$existing_ip" == "$ip" ]]; then
            echo "IP $ip already in whitelist"
            return 0
        fi
    done
    
    # Add new IP
    whitelist+=("$ip")
    save_whitelist whitelist
    echo "Added $ip to whitelist"
}

# Function to remove IP
remove_ip() {
    local ip="$1"
    
    local -a whitelist
    load_whitelist whitelist
    
    local -a new_whitelist=()
    local found=false
    
    for existing_ip in "${whitelist[@]}"; do
        if [[ "$existing_ip" == "$ip" ]]; then
            found=true
            echo "Removing $ip from whitelist"
        else
            new_whitelist+=("$existing_ip")
        fi
    done
    
    if [[ "$found" == true ]]; then
        save_whitelist new_whitelist
    else
        echo "IP $ip not found in whitelist"
        return 1
    fi
}

# Function to list IPs
list_ips() {
    local -a whitelist
    load_whitelist whitelist
    
    if [[ ${#whitelist[@]} -eq 0 ]]; then
        echo "Whitelist is empty"
    else
        echo "Whitelist (${#whitelist[@]} IPs):"
        for i in "${!whitelist[@]}"; do
            echo "  $((i+1)). ${whitelist[$i]}"
        done
    fi
}

# Function to show usage
show_usage() {
    cat << EOF
Usage: $0 {add|remove|list|help} [ip]

Commands:
  add <ip>     - Add IP address to whitelist
  remove <ip>  - Remove IP address from whitelist
  list         - List all whitelisted IPs
  help         - Show this help message

Examples:
  $0 add 192.168.1.100
  $0 remove 10.0.0.5
  $0 list
EOF
}

# Main script
main() {
    local command="${1:-help}"
    
    case "$command" in
        add)
            if [[ -z "$2" ]]; then
                echo "ERROR: Missing IP address"
                show_usage
                exit 1
            fi
            add_ip "$2"
            ;;
        remove)
            if [[ -z "$2" ]]; then
                echo "ERROR: Missing IP address"
                show_usage
                exit 1
            fi
            remove_ip "$2"
            ;;
        list)
            list_ips
            ;;
        help|--help|-h)
            show_usage
            ;;
        *)
            echo "ERROR: Unknown command: $command"
            show_usage
            exit 1
            ;;
    esac
}

# Run main
main "$@"
```

**Usage examples:**

```bash
./whitelist.sh add 192.168.1.100
./whitelist.sh add 10.0.0.5
./whitelist.sh list
./whitelist.sh remove 10.0.0.5
```

**Key features:**

* Uses `local -n` (nameref) for array references (Bash 4.3+)

* Atomic file writes (temp file then `mv`)

* Input validation (IP address format)

* Proper error handling with exit codes

* Case statement for command dispatch
