# 3.1.2 Loops, Conditionals, and Functions: Controlling Script Flow

#### Why Control Flow Matters

Variables store data (from 3.1.1), but **control flow** decides what to do with that data. Conditionals allow your script to make decisions. Loops repeat actions. Functions organize code into reusable blocks. Together, they transform a linear sequence of commands into intelligent automation.

This note covers `if` statements, test operators, `for`/`while` loops, and functions. Note 3.1.1 covered shebangs and variables; note 3.1.3 is the subchapter review.

***

## Part 1: Conditional Statements – Making Decisions

### If-Then-Else Syntax

```bash
# Basic if
if [ condition ]; then
    echo "Condition is true"
fi

# If-else
if [ condition ]; then
    echo "True"
else
    echo "False"
fi

# If-elif-else
if [ condition1 ]; then
    echo "Condition 1 is true"
elif [ condition2 ]; then
    echo "Condition 2 is true"
else
    echo "Neither is true"
fi
```

**Note:** The semicolon `;` before `then` allows writing `then` on the same line. Alternative (more readable for some):

```bash
if [ condition ]
then
    echo "True"
fi
```

### Test Operators – The `[ ]` and `[[ ]]` Commands

Bash has two test constructs:

* `[ condition ]` – POSIX-compatible (slower, fewer features)

* `[[ condition ]]` – Bash-specific (faster, more features, **recommended**)

| Operator               | Meaning                       | Example (using `[[ ]]`)            | <br /> | <br />               | <br /> | <br />             |
| ---------------------- | ----------------------------- | ---------------------------------- | :----- | :------------------- | :----- | :----------------- |
| **String comparison**  | <br />                        | <br />                             | <br /> | <br />               | <br /> | <br />             |
| `=`, `==`              | Equal                         | `[[ "$name" == "Alice" ]]`         | <br /> | <br />               | <br /> | <br />             |
| `!=`                   | Not equal                     | `[[ "$name" != "Bob" ]]`           | <br /> | <br />               | <br /> | <br />             |
| `-z`                   | String is empty (zero length) | `[[ -z "$var" ]]`                  | <br /> | <br />               | <br /> | <br />             |
| `-n`                   | String is not empty           | `[[ -n "$var" ]]`                  | <br /> | <br />               | <br /> | <br />             |
| `<`, `>`               | Lexicographic comparison      | `[[ "apple" < "banana" ]]`         | <br /> | <br />               | <br /> | <br />             |
| **Numeric comparison** | <br />                        | <br />                             | <br /> | <br />               | <br /> | <br />             |
| `-eq`                  | Equal                         | `[[ $count -eq 10 ]]`              | <br /> | <br />               | <br /> | <br />             |
| `-ne`                  | Not equal                     | `[[ $count -ne 0 ]]`               | <br /> | <br />               | <br /> | <br />             |
| `-lt`                  | Less than                     | `[[ $count -lt 100 ]]`             | <br /> | <br />               | <br /> | <br />             |
| `-le`                  | Less than or equal            | `[[ $count -le 50 ]]`              | <br /> | <br />               | <br /> | <br />             |
| `-gt`                  | Greater than                  | `[[ $count -gt 5 ]]`               | <br /> | <br />               | <br /> | <br />             |
| `-ge`                  | Greater than or equal         | `[[ $count -ge 1 ]]`               | <br /> | <br />               | <br /> | <br />             |
| **File tests**         | <br />                        | <br />                             | <br /> | <br />               | <br /> | <br />             |
| `-e`                   | File exists                   | `[[ -e /etc/passwd ]]`             | <br /> | <br />               | <br /> | <br />             |
| `-f`                   | Regular file exists           | `[[ -f /etc/passwd ]]`             | <br /> | <br />               | <br /> | <br />             |
| `-d`                   | Directory exists              | `[[ -d /var/log ]]`                | <br /> | <br />               | <br /> | <br />             |
| `-s`                   | File exists and not empty     | `[[ -s /var/log/syslog ]]`         | <br /> | <br />               | <br /> | <br />             |
| `-r`                   | File is readable              | `[[ -r config.txt ]]`              | <br /> | <br />               | <br /> | <br />             |
| `-w`                   | File is writable              | `[[ -w output.log ]]`              | <br /> | <br />               | <br /> | <br />             |
| `-x`                   | File is executable            | `[[ -x /usr/bin/curl ]]`           | <br /> | <br />               | <br /> | <br />             |
| `-L`                   | File is a symlink             | `[[ -L /bin/bash ]]`               | <br /> | <br />               | <br /> | <br />             |
| **Logical operators**  | <br />                        | <br />                             | <br /> | <br />               | <br /> | <br />             |
| `&&`                   | AND                           | `[[ $age -gt 18 && $age -lt 65 ]]` | <br /> | <br />               | <br /> | <br />             |
| \`                     | <br />                        | \`                                 | OR     | \`\[\[ $day == "Sat" | <br /> | $day == "Sun" ]]\` |
| `!`                    | NOT                           | `[[ ! -f /tmp/lockfile ]]`         | <br /> | <br />               | <br /> | <br />             |

### Practical Condition Examples

```bash
#!/usr/bin/env bash

# Check if argument provided
if [[ $# -eq 0 ]]; then
    echo "Error: No argument provided"
    echo "Usage: $0 <filename>"
    exit 1
fi

# Check if file exists
filename="$1"
if [[ -f "$filename" ]]; then
    echo "File exists: $filename"
else
    echo "Error: File not found: $filename"
    exit 1
fi

# Check if file is readable
if [[ -r "$filename" ]]; then
    echo "File is readable"
else
    echo "Error: Cannot read file (permissions)"
    exit 1
fi

# Numeric comparison
disk_usage=$(df -h / | awk 'NR==2 {print $5}' | sed 's/%//')
if [[ $disk_usage -gt 90 ]]; then
    echo "WARNING: Disk usage is ${disk_usage}%"
elif [[ $disk_usage -gt 75 ]]; then
    echo "INFO: Disk usage is ${disk_usage}%"
else
    echo "OK: Disk usage is ${disk_usage}%"
fi

# String comparison with default value
mode="${1:-production}"
if [[ "$mode" == "production" ]]; then
    echo "Running in PRODUCTION mode"
elif [[ "$mode" == "staging" ]]; then
    echo "Running in STAGING mode"
elif [[ "$mode" == "development" ]]; then
    echo "Running in DEVELOPMENT mode"
else
    echo "Error: Unknown mode '$mode'"
    exit 1
fi
```

### Case Statement (Multiple Pattern Matching)

```bash
#!/usr/bin/env bash

case "$1" in
    start)
        echo "Starting service..."
        /usr/bin/systemctl start myapp
        ;;
    stop)
        echo "Stopping service..."
        /usr/bin/systemctl stop myapp
        ;;
    restart)
        echo "Restarting service..."
        /usr/bin/systemctl restart myapp
        ;;
    status)
        /usr/bin/systemctl status myapp
        ;;
    *)
        echo "Usage: $0 {start|stop|restart|status}"
        exit 1
        ;;
esac
```

**Case pattern features:**

* `*` – Wildcard (any string)

* `?` – Single character

* `[abc]` – Character class

* `a|b` – OR (using `|` inside pattern)

## Part 2: Loops – Repeating Actions

### For Loop – Iterating Over Lists

```bash
# Loop over list of words
for fruit in apple banana orange; do
    echo "Fruit: $fruit"
done

# Loop over range (brace expansion)
for i in {1..10}; do
    echo "Number: $i"
done

# Loop with step (Bash 4+)
for i in {0..100..10}; do
    echo "Multiple of 10: $i"
done

# Loop over array
files=("doc1.txt" "doc2.txt" "doc3.txt")
for file in "${files[@]}"; do
    echo "Processing: $file"
done

# Loop over command output
for user in $(cut -d: -f1 /etc/passwd); do
    echo "User: $user"
done

# Loop with C-style syntax
for ((i=0; i<10; i++)); do
    echo "C-style loop: $i"
done
```

### While Loop – Loop While Condition is True

```bash
# Basic while loop
counter=1
while [[ $counter -le 5 ]]; do
    echo "Count: $counter"
    ((counter++))
done

# Read file line by line
while IFS= read -r line; do
    echo "Line: $line"
done < /etc/hosts

# Read CSV (comma-separated)
while IFS=',' read -r name age city; do
    echo "Name: $name, Age: $age, City: $city"
done < data.csv

# Infinite loop with break
while true; do
    read -p "Enter 'quit' to exit: " input
    if [[ "$input" == "quit" ]]; then
        break
    fi
    echo "You entered: $input"
done
```

### Until Loop – Loop Until Condition is True (Opposite of While)

```bash
counter=1
until [[ $counter -gt 5 ]]; do
    echo "Count: $counter"
    ((counter++))
done
# Same as while [[ $counter -le 5 ]]
```

### Loop Control: Break and Continue

| Command      | Effect                                |
| ------------ | ------------------------------------- |
| `break`      | Exit the loop completely              |
| `break N`    | Exit N levels of nested loops         |
| `continue`   | Skip to next iteration                |
| `continue N` | Skip to next iteration of N levels up |

```bash
# Example with break and continue
for i in {1..10}; do
    if [[ $i -eq 5 ]]; then
        echo "Skipping 5"
        continue
    fi
    if [[ $i -eq 8 ]]; then
        echo "Breaking at 8"
        break
    fi
    echo "Number: $i"
done
# Output: 1,2,3,4,6,7 (skips 5, stops at 8)
```

## Part 3: Functions – Reusable Code Blocks

### Defining and Calling Functions

```bash
# Simple function
greet() {
    echo "Hello, World!"
}

# Call the function
greet

# Function with parameters (inside function, use $1, $2, etc.)
greet_person() {
    echo "Hello, $1! You are $2 years old."
}

greet_person "Alice" 30

# Function with return value (numeric exit code, 0-255)
is_even() {
    local num=$1
    if [[ $((num % 2)) -eq 0 ]]; then
        return 0  # Success (true)
    else
        return 1  # Failure (false)
    fi
}

if is_even 10; then
    echo "10 is even"
fi

# Function that outputs value (use stdout)
get_date() {
    date +%Y-%m-%d
}

today=$(get_date)
echo "Today is $today"
```

### Local Variables

Variables inside functions should be declared `local` to avoid polluting global scope.

```bash
#!/usr/bin/env bash

global_var="I am global"

my_function() {
    local local_var="I am local"
    global_var="Modified global"
    echo "Inside function: $local_var"
}

my_function
echo "Outside: $global_var"
echo "Outside local_var: $local_var"  # Empty (not accessible)
```

### Function Return Values

Functions can return:

1. **Numeric exit codes** (`return N`) – for success/failure
2. **String output** (`echo`) – for data

```bash
# Return numeric exit code (for condition checking)
file_exists() {
    [[ -f "$1" ]]
    return $?  # Returns 0 if exists, 1 if not
}

if file_exists "/etc/passwd"; then
    echo "File exists"
fi

# Return string output (for data)
get_ip() {
    hostname -I | awk '{print $1}'
}

server_ip=$(get_ip)
echo "Server IP: $server_ip"
```

### Function Libraries

Create reusable function libraries:

```bash
# lib/utils.sh
#!/usr/bin/env bash

log_info() {
    echo "[INFO] $(date '+%Y-%m-%d %H:%M:%S') - $*"
}

log_error() {
    echo "[ERROR] $(date '+%Y-%m-%d %H:%M:%S') - $*" >&2
}

check_command() {
    if ! command -v "$1" &>/dev/null; then
        log_error "Command '$1' not found"
        return 1
    fi
    return 0
}
```

```bash
# main.sh
#!/usr/bin/env bash

# Source the library
source /path/to/lib/utils.sh

log_info "Starting script"
check_command "curl" || exit 1
log_info "Script completed"
```

## Part 4: Combining Conditionals, Loops, and Functions

### Example: Backup Script with Error Handling

```bash
#!/usr/bin/env bash

# Function to log messages
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*"
}

# Function to check if backup directory exists
check_backup_dir() {
    local backup_dir="$1"
    if [[ ! -d "$backup_dir" ]]; then
        log "ERROR: Backup directory $backup_dir does not exist"
        return 1
    fi
    if [[ ! -w "$backup_dir" ]]; then
        log "ERROR: Backup directory $backup_dir is not writable"
        return 1
    fi
    return 0
}

# Function to perform backup
perform_backup() {
    local source_dir="$1"
    local backup_dir="$2"
    local timestamp=$(date +%Y%m%d_%H%M%S)
    local backup_name="backup_${timestamp}.tar.gz"
    
    log "Starting backup of $source_dir"
    
    if [[ ! -d "$source_dir" ]]; then
        log "ERROR: Source directory $source_dir does not exist"
        return 1
    fi
    
    cd "$source_dir" || return 1
    tar -czf "${backup_dir}/${backup_name}" . 2>/dev/null
    
    if [[ $? -eq 0 ]]; then
        log "SUCCESS: Backup created at ${backup_dir}/${backup_name}"
        echo "${backup_dir}/${backup_name}"
        return 0
    else
        log "ERROR: Backup failed"
        return 1
    fi
}

# Main script
main() {
    local source="${1:-/var/www}"
    local backup="${2:-/backup}"
    
    log "=== Backup Script Started ==="
    
    if ! check_backup_dir "$backup"; then
        exit 1
    fi
    
    if backup_file=$(perform_backup "$source" "$backup"); then
        log "Backup completed: $backup_file"
    else
        log "Backup failed"
        exit 1
    fi
    
    log "=== Backup Script Finished ==="
}

# Run main function with all arguments
main "$@"
```

### Example: Process Monitoring Script

```bash
#!/usr/bin/env bash

# Function to check if process is running
is_process_running() {
    local process_name="$1"
    pgrep -x "$process_name" >/dev/null 2>&1
}

# Function to restart process
restart_process() {
    local process_name="$1"
    local service_name="$2"
    
    echo "Restarting $process_name..."
    if systemctl restart "$service_name" 2>/dev/null; then
        echo "SUCCESS: $process_name restarted"
        return 0
    else
        echo "ERROR: Failed to restart $process_name"
        return 1
    fi
}

# Main monitoring loop
check_processes() {
    local processes=(
        "nginx:nginx"
        "mysql:mysql"
        "redis:redis-server"
    )
    
    for entry in "${processes[@]}"; do
        process_name="${entry%:*}"
        service_name="${entry#*:}"
        
        if is_process_running "$process_name"; then
            echo "✓ $process_name is running"
        else
            echo "✗ $process_name is NOT running"
            restart_process "$process_name" "$service_name"
        fi
    done
}

# Run check
check_processes
```

## Quick Task: Scripting Practice

*Write scripts that use conditionals, loops, and functions.*

1. Write a script that takes a number as argument and prints whether it's positive, negative, or zero.
2. Write a script that loops through all `.log` files in `/var/log` and prints their names and sizes.
3. Write a function `is_valid_ip` that checks if a string is a valid IPv4 address.
4. Write a script that uses a case statement to handle start/stop/restart/status for a service.

## Summary Table: Conditionals and Loops

| Construct       | Syntax                                                      | Use Case              |
| --------------- | ----------------------------------------------------------- | --------------------- |
| `if`            | `if [[ condition ]]; then ... fi`                           | Conditional execution |
| `if-else`       | `if [[ cond ]]; then ... else ... fi`                       | Two-way branch        |
| `if-elif-else`  | `if [[ c1 ]]; then ... elif [[ c2 ]]; then ... else ... fi` | Multi-way branch      |
| `case`          | `case "$var" in pattern) ... ;; esac`                       | Pattern matching      |
| `for` (list)    | `for item in list; do ... done`                             | Iterate over list     |
| `for` (C-style) | `for ((i=0; i<10; i++)); do ... done`                       | Numeric iteration     |
| `while`         | `while [[ condition ]]; do ... done`                        | Loop while true       |
| `until`         | `until [[ condition ]]; do ... done`                        | Loop until true       |
| `break`         | `break`                                                     | Exit loop             |
| `continue`      | `continue`                                                  | Skip iteration        |

### Test Operators Quick Reference

| Category    | Operators                                      | <br /> | <br />  |
| ----------- | ---------------------------------------------- | :----- | :------ |
| **Strings** | `=`, `==`, `!=`, `-z`, `-n`                    | <br /> | <br />  |
| **Numbers** | `-eq`, `-ne`, `-lt`, `-le`, `-gt`, `-ge`       | <br /> | <br />  |
| **Files**   | `-e`, `-f`, `-d`, `-s`, `-r`, `-w`, `-x`, `-L` | <br /> | <br />  |

### Function Syntax

| Component         | Syntax                           |
| ----------------- | -------------------------------- |
| Define function   | `name() { commands; }`           |
| Call function     | `name` or `name arg1 arg2`       |
| Local variable    | `local var=value`                |

| Access parameters | `$1`, `$2`, `$@` inside function |
