### Data Types Quick Reference

| Type | Example | Mutable | Best for |
|------|---------|---------|----------|
| `str` | `"hello"` | No | Text, paths |

### f-strings

```python
name = "Alice"
cpu  = 87.567
port = 8080

f"Hello {name}"           # Hello Alice
f"CPU: {cpu:.1f}%"        # CPU: 87.6%
f"CPU: {cpu:.0f}%"        # CPU: 88%
f"{name.upper()}"         # ALICE
f"{name!r}"               # 'Alice'   ← repr, good for debugging
f"Port: {port:05d}"       # Port: 08080 ← zero-padded
```

## Bridge Concepts

Concepts used in the notes but needing extra clarification:

| Concept | Explanation |
|---------|-------------|
| `venv` | Python virtual environment. Isolates dependencies per project. Just a directory with a Python binary and `lib/` folder. |
| `pip freeze` | Outputs all installed packages with exact versions. Use to generate `requirements.txt`. |
| `__name__ == "__main__"` | Python sets `__name__` to `"__main__"` only when the file is run directly, not imported. Covered fully in 9.4.2. |
| `defaultdict(int)` | A dict that returns `0` (or other default) instead of `KeyError` for missing keys. Covered in 9.1.4. |
| `yaml.safe_load()` | Safe YAML parser — never executes embedded code. Always use instead of `yaml.load()`. |
| `json.load()` vs `json.loads()` | `load(f)` reads from file object; `loads(s)` parses a string. Same pattern for `dump`/`dumps`. |
| `strip()` | Removes leading and trailing whitespace (including `\n`). Use when reading lines from files. |
| `os.makedirs(exist_ok=True)` | Creates all intermediate directories. `exist_ok=True` prevents error if directory already exists. |
| `shutil.copy2` | Copies file AND metadata (timestamps, permissions). Prefer over `shutil.copy` for backups. |
| `re.compile()` | Pre-compiles regex pattern. Faster when the same pattern is used hundreds/thousands of times. |
| f-string `:.1f` | Float format: 1 decimal place. `:05d` = zero-pad integer to 5 digits. `!r` = repr format. |
| Walrus `:=` | Python 3.8+. Assigns and returns in one expression. `if m := re.search(...):` — no need for separate assignment line. |
| `match/case` | Python 3.10+. Structural pattern matching. `match event: case {"type": "ADDED"}: ...`. Destructures dicts, lists, and types. |
| `tempfile.NamedTemporaryFile` | Creates a temp file with a name accessible by other processes. `delete=True` (default) removes it on close. Use for passing temp configs to CLI tools. |
| `tempfile.TemporaryDirectory` | Creates a temp directory, auto-removed when the context manager exits. Use as a workspace for builds and tests. |

## Interview Questions

### Question 1

**Scenario:** A 10 GB Nginx access log must be analyzed to count occurrences of each HTTP status code (200, 404, 500...).

**Question:** Why is `f.read()` a bad choice here? Write memory-efficient code using a dictionary.

**Answer:**

`f.read()` loads the entire 10 GB into RAM — likely causing an OOM kill or swap thrashing. The fix is to read one line at a time:

```python
from collections import defaultdict

def count_status_codes(log_file):
    counts = defaultdict(int)

    with open(log_file, 'r') as f:
        for line in f:                  # reads ONE line at a time
            parts = line.split()
            if len(parts) >= 9:
                status = parts[8]       # Nginx combined log format: field 9
                counts[status] += 1

    return dict(sorted(counts.items()))

# Usage
for status, count in count_status_codes('/var/log/nginx/access.log').items():
    print(f"HTTP {status}: {count:,} requests")
```

Memory used: ~one line (~200 bytes) regardless of file size.

### Question 2

**Scenario:** A script reads a YAML config file. If the file doesn't exist, it creates it with defaults.

**Question:** Write a safe `load_or_create_config` function.

**Answer:**

```python
import yaml, os

DEFAULT_CONFIG = {
    'server':   {'host': '0.0.0.0', 'port': 8080},
    'database': {'host': 'localhost', 'port': 5432, 'name': 'myapp'},
    'logging':  {'level': 'INFO'}
}

def load_or_create_config(config_path):
    if os.path.exists(config_path):
        with open(config_path, 'r') as f:
            config = yaml.safe_load(f)         # safe — never yaml.load()
        print(f"Loaded config from {config_path}")
        return config
    else:
        os.makedirs(os.path.dirname(config_path), exist_ok=True)
        with open(config_path, 'w') as f:
            yaml.dump(DEFAULT_CONFIG, f, default_flow_style=False)
        print(f"Created default config at {config_path}")
        return DEFAULT_CONFIG.copy()

config = load_or_create_config('/etc/myapp/config.yaml')
print(f"Server port: {config['server']['port']}")
```

### Question 3

**Scenario:** A function receives deeply nested dicts from Kubernetes API responses. Accessing `data['status']['containerStatuses'][0]['restartCount']` crashes on pods that haven't started yet.

**Question:** Write a `safe_get` function that returns `None` (or a default) for any missing key path.

**Answer:**

```python
def safe_get(data, *keys, default=None):
    """Safely traverse nested dicts/lists.

    Usage:
        safe_get(pod, 'status', 'containerStatuses', 0, 'restartCount')
    """
    current = data
    for key in keys:
        try:
            current = current[key]
        except (KeyError, IndexError, TypeError):
            return default
    return current

# Examples
pod = {'status': {'phase': 'Pending'}}  # no containerStatuses yet

restarts = safe_get(pod, 'status', 'containerStatuses', 0, 'restartCount', default=0)
print(restarts)   # 0 (not a crash)

phase = safe_get(pod, 'status', 'phase', default='Unknown')
print(phase)      # Pending
```

### Question 4

**Scenario:** Find all Python files in a directory tree, count only code lines (skip blank lines and `#` comments), then print top 5 files by code size.

**Question:** Write `analyze_python_files(directory)` using `os.walk`.

**Answer:**

```python
import os

def count_code_lines(filepath):
    """Count non-blank, non-comment lines"""
    code = 0
    with open(filepath, 'r', encoding='utf-8', errors='ignore') as f:
        for line in f:
            stripped = line.strip()
            if stripped and not stripped.startswith('#'):
                code += 1
    return code

def analyze_python_files(directory):
    results = []
    skip_dirs = {'venv', 'env', '.git', '__pycache__', '.tox', 'node_modules'}

    for root, dirs, files in os.walk(directory):
        dirs[:] = [d for d in dirs if d not in skip_dirs]  # prune in-place

        for file in files:
            if file.endswith('.py'):
                path = os.path.join(root, file)
                results.append({'path': path, 'lines': count_code_lines(path)})

    results.sort(key=lambda x: x['lines'], reverse=True)

    print(f"Top 5 Python files by code size:")
    for r in results[:5]:
        print(f"  {r['lines']:4d} lines  {r['path']}")

    return results

analyze_python_files('.')
```

### Question 5

**Scenario:** Extract all IPs from an Nginx log that caused 404 errors, count occurrences, show top 10.

**Question:** Write the function using `re` and a dictionary.

**Answer:**

```python
import re
from collections import defaultdict

def analyze_404_ips(log_file):
    # Nginx combined log: IP - - [time] "REQUEST" STATUS SIZE
    pattern = re.compile(r'^(\d{1,3}(?:\.\d{1,3}){3}) .+ " (\d{3}) ')
    ip_counts = defaultdict(int)
    total = 0

    with open(log_file, 'r') as f:
        for line in f:
            m = pattern.match(line)
            if m and m.group(2) == '404':
                ip_counts[m.group(1)] += 1
                total += 1

    top10 = sorted(ip_counts.items(), key=lambda x: x[1], reverse=True)[:10]

    print(f"Total 404 errors: {total}")
    print(f"Unique IPs:       {len(ip_counts)}")
    print("\nTop 10 offenders:")
    for ip, count in top10:
        print(f"  {ip:<18} {count:>5} requests")

analyze_404_ips('/var/log/nginx/access.log')
```
