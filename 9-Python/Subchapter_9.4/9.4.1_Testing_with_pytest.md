### 9.4.1 Testing with pytest: Building Reliable Automation

#### Why Testing Matters

Automation scripts that break in production are worse than no automation at all. Testing ensures:
- **Correctness** – Functions do what they claim
- **Reliability** – Changes don't break existing behavior
- **Confidence** – Deploy with peace of mind
- **Documentation** – Tests show how code should work

`pytest` is the industry standard for Python testing – simple, powerful, and widely used.

This note covers pytest. Note 9.4.2 covers production-ready patterns; note 9.4.3 is the subchapter review with final exam.

**Backward references:** Functions from 9.1.1; error handling from 9.3.1; file I/O from 9.1.2.

### Test Discovery Rules

pytest automatically finds tests:
- Files named `test_*.py` or `*_test.py`
- Functions named `test_*`
- Classes named `Test*` with methods named `test_*`

## Part 2: Basic Tests with assert

### Simple Test Example

```python
# calculator.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def multiply(a, b):
    return a * b

def divide(a, b):
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b
```

```python
# tests/test_calculator.py
from myapp.calculator import add, subtract, multiply, divide

def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0
    assert add(0, 0) == 0

def test_subtract():
    assert subtract(10, 5) == 5
    assert subtract(0, 5) == -5

def test_multiply():
    assert multiply(3, 4) == 12
    assert multiply(-2, 3) == -6

def test_divide():
    assert divide(10, 2) == 5
    assert divide(9, 3) == 3
```

## Part 4: Fixtures – Setup and Teardown

Fixtures provide reusable test setup.

```python
import pytest
import tempfile
import os

@pytest.fixture
def temp_file():
    """Create a temporary file for testing"""
    fd, path = tempfile.mkstemp()
    yield path  # Provide the fixture value
    # Teardown (runs after test)
    os.close(fd)
    os.unlink(path)

def test_write_read(temp_file):
    # Write to temp file
    with open(temp_file, 'w') as f:
        f.write("test data")
    
    # Read from temp file
    with open(temp_file, 'r') as f:
        content = f.read()
    
    assert content == "test data"
```

### Fixture Scopes

| Scope | Runs | Use Case |
|-------|------|----------|
| `function` (default) | Once per test | Most fixtures |
| `class` | Once per test class | Class-level setup |
| `module` | Once per module | Module-level resources |
| `session` | Once per test session | Expensive resources |

```python
import pytest

@pytest.fixture(scope="session")
def db_connection():
    """Create database connection once for all tests"""
    print("\nSetting up database connection")
    conn = create_db_connection()
    yield conn
    print("\nClosing database connection")
    conn.close()

@pytest.fixture(scope="module")
def test_data():
    """Load test data once per module"""
    print("\nLoading test data")
    return {"user": "alice", "age": 30}
```

### Multiple Fixtures

```python
@pytest.fixture
def user_data():
    return {"name": "Alice", "email": "alice@example.com"}

@pytest.fixture
def user_list(user_data):
```

### Built-in Fixtures

```python
import pytest
import tempfile
```

## Part 5: Parametrize – Testing Multiple Inputs

`@pytest.mark.parametrize` runs the same test with different inputs.

```python
import pytest

@pytest.mark.parametrize("a, b, expected", [
    (1, 2, 3),
    (0, 0, 0),
    (-1, 1, 0),
    (100, 200, 300),
])
def test_add(a, b, expected):
    assert add(a, b) == expected

# Multiple parameters
@pytest.mark.parametrize("a", [1, 2, 3])
@pytest.mark.parametrize("b", [10, 20, 30])
def test_add_multiple(a, b):
    result = add(a, b)
    assert result == a + b

# Parametrize with ids
@pytest.mark.parametrize("a, b, expected", [
    (1, 2, 3),
    (0, 0, 0),
    (-1, 1, 0),
], ids=["positive", "zero", "negative"])
def test_add_with_ids(a, b, expected):
    assert add(a, b) == expected
```

## Part 6: Mocking with unittest.mock

Mocking replaces real objects with test doubles.

```python
from unittest.mock import Mock, patch
import pytest
import requests

def get_user_data(user_id):
    """Function that calls external API"""
    response = requests.get(f"https://api.example.com/users/{user_id}")
    return response.json()

# Using patch as a decorator
@patch('myapp.api.requests.get')
def test_get_user_data(mock_get):
    # Configure mock
    mock_response = Mock()
    mock_response.json.return_value = {"id": 1, "name": "Alice"}
    mock_get.return_value = mock_response
    
    # Call function
    result = get_user_data(1)
    
    # Assert
    assert result["name"] == "Alice"
    mock_get.assert_called_once_with("https://api.example.com/users/1")

# Using patch as a context manager
def test_with_context_manager():
    with patch('myapp.api.requests.get') as mock_get:
        mock_response = Mock()
        mock_response.json.return_value = {"id": 1}
        mock_get.return_value = mock_response
        
        result = get_user_data(1)
        assert result["id"] == 1
```

### Mocking Environment Variables

```python
import os
from unittest.mock import patch

def get_db_config():
    return {
        'host': os.environ.get('DB_HOST', 'localhost'),
        'port': int(os.environ.get('DB_PORT', '5432'))
    }

@patch.dict(os.environ, {'DB_HOST': 'prod.example.com', 'DB_PORT': '5433'})
def test_get_db_config_with_env():
    config = get_db_config()
    assert config['host'] == 'prod.example.com'
    assert config['port'] == 5433

def test_get_db_config_default():
    with patch.dict(os.environ, {}, clear=True):
        config = get_db_config()
        assert config['host'] == 'localhost'
        assert config['port'] == 5432
```

### Mocking File Operations

```python
from unittest.mock import mock_open, patch

def read_config(filepath):
    with open(filepath, 'r') as f:
        return f.read()

@patch("builtins.open", new_callable=mock_open, read_data='{"key": "value"}')
def test_read_config(mock_file):
    content = read_config("config.json")
    assert content == '{"key": "value"}'
    mock_file.assert_called_once_with("config.json", 'r')
```

## Part 7: Testing Automation Scripts

### Example: Testing a File Processor

```python
# file_processor.py
from pathlib import Path

def process_log_file(filepath):
    """Count error lines in log file"""
    error_count = 0
    path = Path(filepath)
    
    if not path.exists():
        raise FileNotFoundError(f"File not found: {filepath}")
    
    with open(filepath, 'r') as f:
        for line in f:
            if 'ERROR' in line:
                error_count += 1
    
    return error_count
```

```python
# tests/test_file_processor.py
import pytest
from pathlib import Path
from file_processor import process_log_file

@pytest.fixture
def sample_log(tmp_path):
    """Create a sample log file for testing"""
    log_content = """INFO: Starting application
ERROR: Database connection failed
INFO: Retrying connection
ERROR: Timeout after 30s
INFO: Shutting down"""
    
    log_file = tmp_path / "app.log"
    log_file.write_text(log_content)
    return log_file

def test_process_log_file(sample_log):
    error_count = process_log_file(sample_log)
    assert error_count == 2

def test_process_log_file_not_found():
    with pytest.raises(FileNotFoundError) as exc_info:
        process_log_file("/nonexistent/file.log")
    assert "File not found" in str(exc_info.value)

@pytest.mark.parametrize("content, expected", [
    ("", 0),
    ("INFO: OK", 0),
    ("ERROR: Failed", 1),
    ("ERROR: First\nERROR: Second", 2),
])
def test_process_log_file_parametrize(tmp_path, content, expected):
    log_file = tmp_path / "test.log"
    log_file.write_text(content)
    assert process_log_file(log_file) == expected
```

### Example: Testing Kubernetes Client

```python
# k8s_client.py
import subprocess
import json

def get_pods(namespace='default'):
    """Get pods using kubectl"""
    cmd = ['kubectl', 'get', 'pods', '-n', namespace, '-o', 'json']
    result = subprocess.run(cmd, capture_output=True, text=True)
    
    if result.returncode != 0:
        raise Exception(f"kubectl failed: {result.stderr}")
    
    data = json.loads(result.stdout)
    return [item['metadata']['name'] for item in data.get('items', [])]
```

```python
# tests/test_k8s_client.py
import pytest
import json
from unittest.mock import patch, Mock
from k8s_client import get_pods

@patch('subprocess.run')
def test_get_pods_success(mock_run):
    # Mock kubectl output
    mock_result = Mock()
    mock_result.returncode = 0
    mock_result.stdout = json.dumps({
        'items': [
            {'metadata': {'name': 'pod-1'}},
            {'metadata': {'name': 'pod-2'}}
        ]
    })
    mock_run.return_value = mock_result
    
    pods = get_pods('default')
    
    assert pods == ['pod-1', 'pod-2']
    mock_run.assert_called_once()
    call_args = mock_run.call_args[0][0]
    assert call_args == ['kubectl', 'get', 'pods', '-n', 'default', '-o', 'json']

@patch('subprocess.run')
def test_get_pods_failure(mock_run):
    mock_result = Mock()
    mock_result.returncode = 1
    mock_result.stderr = "Error: connection refused"
    mock_run.return_value = mock_result
    
    with pytest.raises(Exception) as exc_info:
        get_pods('default')
    
    assert "kubectl failed" in str(exc_info.value)
```

## Part 8: Running Tests

### Command-Line Options

```bash
# Run all tests
pytest

# Run specific file
pytest tests/test_calculator.py

# Run specific test function
pytest tests/test_calculator.py::test_add

# Run tests matching keyword
pytest -k "add or subtract"

# Run with verbose output
pytest -v

# Stop on first failure
pytest -x

# Run with coverage
pytest --cov=myapp tests/

# Run with verbose and coverage
pytest -v --cov=myapp --cov-report=html tests/

# Show print output (even for passing tests)
pytest -s

# Run only slow tests (marked)
pytest -m slow
```

## Quick Task: Write Tests for a Function

*Write tests for a function that validates Kubernetes manifest files.*

```python
# validator.py
import yaml
from pathlib import Path

def validate_manifest(filepath):
    """Validate a Kubernetes manifest YAML file"""
    path = Path(filepath)
    
    if not path.exists():
        raise FileNotFoundError(f"File not found: {filepath}")
    
    if not filepath.endswith(('.yaml', '.yml')):
        raise ValueError("File must be .yaml or .yml")
    
    try:
        with open(filepath, 'r') as f:
            data = yaml.safe_load(f)
    except yaml.YAMLError as e:
        raise ValueError(f"Invalid YAML: {e}")
    
    # Check required fields
    if 'apiVersion' not in data:
        raise ValueError("Missing apiVersion")
    if 'kind' not in data:
        raise ValueError("Missing kind")
    if 'metadata' not in data or 'name' not in data['metadata']:
        raise ValueError("Missing metadata.name")
    
    return True
```

```python
# tests/test_validator.py
import pytest
import yaml
from pathlib import Path
from validator import validate_manifest

@pytest.fixture
def valid_manifest(tmp_path):
    manifest = {
        'apiVersion': 'v1',
        'kind': 'Pod',
        'metadata': {'name': 'test-pod'},
        'spec': {'containers': [{'name': 'nginx', 'image': 'nginx'}]}
    }
    filepath = tmp_path / 'valid.yaml'
    with open(filepath, 'w') as f:
        yaml.dump(manifest, f)
    return filepath

@pytest.fixture
def invalid_yaml(tmp_path):
    filepath = tmp_path / 'invalid.yaml'
    filepath.write_text("invalid: yaml: content: [")
    return filepath

def test_valid_manifest(valid_manifest):
    assert validate_manifest(valid_manifest) is True

def test_file_not_found():
    with pytest.raises(FileNotFoundError):
        validate_manifest("/nonexistent.yaml")

def test_wrong_extension(tmp_path):
    filepath = tmp_path / 'test.txt'
    filepath.write_text("content")
    with pytest.raises(ValueError) as exc_info:
        validate_manifest(filepath)
    assert "File must be .yaml or .yml" in str(exc_info.value)

def test_invalid_yaml(invalid_yaml):
    with pytest.raises(ValueError) as exc_info:
        validate_manifest(invalid_yaml)
    assert "Invalid YAML" in str(exc_info.value)

@pytest.mark.parametrize("missing_field", ['apiVersion', 'kind'])
def test_missing_field(valid_manifest, missing_field, tmp_path):
    import yaml
    with open(valid_manifest, 'r') as f:
        data = yaml.safe_load(f)
    del data[missing_field]
    
    new_file = tmp_path / 'missing.yaml'
    with open(new_file, 'w') as f:
        yaml.dump(data, f)
    
    with pytest.raises(ValueError) as exc_info:
        validate_manifest(new_file)
    assert missing_field in str(exc_info.value)
```

## Summary Table: pytest

### Test Discovery

| Pattern | Matches |
|---------|---------|
| `test_*.py` | Test files |
| `*_test.py` | Test files |
| `def test_*()` | Test functions |
| `class Test*` | Test classes |

### Common Assertions

| Assertion | Use |
|-----------|-----|
| `assert a == b` | Equality |
| `assert a != b` | Inequality |
| `assert a is None` | None check |
| `assert item in list` | Membership |
| `assert a > b` | Comparison |
| `assert condition` | Boolean |

### Fixture Scopes

| Scope | Runs |
|-------|------|
| `function` | Once per test (default) |
| `class` | Once per test class |
| `module` | Once per module |
| `session` | Once per test session |

### pytest Command-Line Options

| Option | Purpose |
|--------|---------|
| `-v` | Verbose output |
| `-x` | Stop on first failure |
| `-k EXPR` | Run tests matching expression |
| `-m MARK` | Run tests with marker |
| `-s` | Show print output |
| `--cov=PATH` | Coverage report |
| `--lf` | Run only failed tests |
