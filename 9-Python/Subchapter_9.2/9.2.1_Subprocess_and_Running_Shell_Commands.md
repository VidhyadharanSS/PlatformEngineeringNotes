## Why Subprocess Matters

Python scripts for platform engineering almost always need to call external tools:
- `kubectl get pods` — query Kubernetes
- `docker build` — build images  
- `aws s3 sync` — cloud operations
- `helm upgrade` — deploy charts
- `git log` — query repository state

The `subprocess` module is the safe, modern way to run these commands from Python, capturing output, handling errors, and building automation on top of CLI tools.

> **Warning:** Never call `subprocess.run(cmd, shell=True)` with user-supplied input. Shell metacharacters (`;`, `&&`, `$()`, backticks) are interpreted by `/bin/sh` — classic shell injection. Default to `shell=False` and pass the command as a **list**: `subprocess.run(["kubectl", "get", "pods"])`.

> **Tip:** `subprocess.run(..., check=True, capture_output=True, text=True)` is the modern one-liner that replaces almost every old `subprocess.Popen` pattern. `check=True` raises `CalledProcessError` on non-zero exit, and `text=True` returns strings instead of bytes.

## Part 1: Why Not `os.system()`?

```python
import os

# ❌ DANGEROUS AND LIMITED — never use in production
os.system('ls -la')  # output goes directly to terminal, can't capture

# Command injection vulnerability:
user_input = "file.txt; rm -rf /"
os.system(f"ls {user_input}")   # ← runs BOTH ls AND rm -rf /!
# The shell expands the semicolon → catastrophic

# No easy return code check
# No way to capture output
# No timeout
```

Use `subprocess.run()` instead — always.

## Part 2: `subprocess.run()` — The Modern Way

### Basic Usage

```python
import subprocess

# Run command, wait for completion
# Pass command as a LIST — never concatenate strings into a single shell command
result = subprocess.run(['ls', '-la'])

# Check return code (0 = success, anything else = error)
if result.returncode == 0:
    print("Success!")
else:
    print(f"Failed with code {result.returncode}")
```

> **Why a list, not a string?** `subprocess.run(['ls', '-la'])` passes `ls` and `-la` as separate arguments directly to the OS — no shell interpretation. With a list, there is **no way** for user input to inject extra commands. If you pass a string and `shell=True`, the shell interprets it — and an attacker can inject `;rm -rf /`.

### Capturing Output

```python
import subprocess

# capture_output=True captures both stdout and stderr
# text=True returns strings instead of bytes
result = subprocess.run(['ls', '-la'], capture_output=True, text=True)

print(result.stdout)    # the actual output
print(result.stderr)    # any error messages
print(result.returncode) # 0 = success

# capture_output=True is shorthand for:
# stdout=subprocess.PIPE, stderr=subprocess.PIPE
```

> **`capture_output=True` vs `stdout=subprocess.PIPE`:** They are identical. `capture_output=True` was added in Python 3.7 as a convenience shorthand. Both set `stdout=PIPE, stderr=PIPE`.

### Feeding stdin with `input=`

> **Why this matters:** Many DevOps tools accept input via stdin: `kubectl apply -f -` reads YAML from stdin, `docker login --password-stdin` reads passwords safely, `gpg --import` reads keys from stdin. Without `input=`, engineers resort to `shell=True` with `echo |` — a security risk.

```python
import subprocess
import yaml

# kubectl apply -f - (read YAML from stdin instead of a file)
manifest = {
    'apiVersion': 'v1',
    'kind': 'ConfigMap',
    'metadata': {'name': 'my-config', 'namespace': 'default'},
    'data': {'key': 'value'}
}

result = subprocess.run(
    ['kubectl', 'apply', '-f', '-'],   # '-' means read from stdin
    input=yaml.dump(manifest),          # ← fed to the command's stdin
    capture_output=True,
    text=True,                          # input must be str when text=True
    timeout=30
)
print(result.stdout)  # "configmap/my-config configured"

# docker login --password-stdin (avoids password in command line / ps output)
result = subprocess.run(
    ['docker', 'login', '--username', 'myuser', '--password-stdin', 'ghcr.io'],
    input='my-secret-token\n',         # password via stdin, NOT in args
    capture_output=True,
    text=True
)

# Feed binary data (text=False, input must be bytes)
result = subprocess.run(
    ['gpg', '--import'],
    input=key_bytes,                   # bytes when text=False (default)
    capture_output=True
)

# Generate YAML dynamically and apply without temp files
import json
pods_json = json.dumps({"items": [...]})
result = subprocess.run(
    ['jq', '.items[].metadata.name'],
    input=pods_json,
    capture_output=True, text=True
)
```

> **`input=` vs `stdin=`:** `input=` is a convenience that passes a string/bytes directly. `stdin=subprocess.PIPE` + `proc.communicate(input=data)` is the `Popen` equivalent. For `subprocess.run()`, always use `input=` — it's simpler and handles the pipe setup internally.

### Checking Return Code

```python
import subprocess

# Method 1: Check manually
result = subprocess.run(['ls', '/nonexistent'], capture_output=True, text=True)
if result.returncode != 0:
    print(f"Command failed: {result.stderr}")

# Method 2: check=True — raises CalledProcessError on non-zero exit code
try:
    subprocess.run(['ls', '/nonexistent'], check=True, capture_output=True, text=True)
except subprocess.CalledProcessError as e:
    print(f"Command failed with code {e.returncode}")
    print(f"stderr: {e.stderr}")
```

### Timeout

```python
import subprocess

try:
    result = subprocess.run(
        ['kubectl', 'rollout', 'status', 'deployment/myapp'],
        capture_output=True,
        text=True,
        timeout=300     # 5 minutes max
    )
except subprocess.TimeoutExpired:
    print("kubectl rollout timed out after 300 seconds")
```

### Custom Working Directory and Environment

```python
import subprocess, os

# Run in a different directory
result = subprocess.run(
    ['git', 'log', '--oneline', '-5'],
    capture_output=True,
    text=True,
    cwd='/path/to/repo'    # change working directory for this command
)

# Custom environment variables
env = os.environ.copy()    # start with current environment
env['KUBECONFIG'] = '/home/user/.kube/prod-config'
env['DEBUG'] = 'true'

result = subprocess.run(
    ['kubectl', 'get', 'pods'],
    capture_output=True,
    text=True,
    env=env
)

# Suppress all output (neither capture NOR print)
result = subprocess.run(
    ['apt-get', 'update'],
    stdout=subprocess.DEVNULL,    # discard stdout
    stderr=subprocess.DEVNULL,    # discard stderr
)
```

> **`subprocess.DEVNULL`:** Redirects output to `/dev/null` — the command runs but you never see or store its output. Useful when you only care about the return code (success/failure), not the output.

### Complete Helper Function

```python
import subprocess
from typing import Optional

def run_command(
    cmd: list[str],
    timeout: int = 30,
    check: bool = False,
    cwd: Optional[str] = None,
    env: Optional[dict] = None
) -> tuple[bool, str, str]:
    """
    Run a command and return (success, stdout, stderr).

    Args:
        cmd:     Command as list, e.g. ['kubectl', 'get', 'pods']
        timeout: Timeout in seconds
        check:   If True, raise on non-zero exit
        cwd:     Working directory
        env:     Environment variables

    Returns:
        (success: bool, stdout: str, stderr: str)
    """
    try:
        result = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            timeout=timeout,
            check=check,
            cwd=cwd,
            env=env
        )
        return result.returncode == 0, result.stdout, result.stderr

    except subprocess.TimeoutExpired:
        return False, "", f"Command timed out after {timeout}s"
    except subprocess.CalledProcessError as e:
        return False, e.stdout or "", e.stderr or ""
    except FileNotFoundError:
        return False, "", f"Command not found: {cmd[0]}"
```

## Part 3: `shlex.split()` — Safely Split Shell Command Strings

> **Problem:** Sometimes you receive a command as a single string (from config, user input, or an API response) but need a list for `subprocess.run()`. You cannot just split on spaces — quoted arguments like `"my file.txt"` would break.

```python
import shlex

# WRONG — splits on space, breaks quoted args
cmd_string = "kubectl get pods --field-selector 'status.phase=Running'"
subprocess.run(cmd_string.split())  # ❌ 'status.phase=Running' becomes wrong token

# CORRECT — shlex.split handles quotes, escapes correctly
cmd_string = "kubectl get pods --field-selector 'status.phase=Running'"
cmd_list   = shlex.split(cmd_string)
# ['kubectl', 'get', 'pods', '--field-selector', 'status.phase=Running']
subprocess.run(cmd_list)   # ✅

# Also useful for reverse: list → shell-safe string (for display/logging)
import shlex
cmd = ['ls', '-la', '/path with spaces/file.txt']
print(shlex.join(cmd))  # ls -la '/path with spaces/file.txt'

# Real-world: parse a command from a YAML config file
import yaml
with open('tasks.yaml') as f:
    config = yaml.safe_load(f)
cmd = shlex.split(config['health_check_command'])   # "curl -f http://localhost:8080/health"
subprocess.run(cmd, timeout=10)
```

## Part 4: Running Shell Commands with Pipes

### Using `shell=True` — Use with Extreme Caution

```python
import subprocess

# shell=True lets you use bash built-ins and pipe operators
# BUT: only use this with hardcoded strings, NEVER with user input
result = subprocess.run(
    "grep -c ERROR /var/log/syslog",
    shell=True,
    capture_output=True,
    text=True
)
print(f"Error count: {result.stdout.strip()}")

# ❌ NEVER with user input — command injection:
user_input = "file.txt; rm -rf /"
subprocess.run(f"cat {user_input}", shell=True)  # catastrophic!
```

### Safer Alternative: Chain Popen Processes

> **Why `p1.stdout.close()` is needed:** When `p1` finishes writing, the read end of the pipe must be closed so `p2` (reading from it) receives EOF and knows input is done. If you don't close it, `p2` hangs waiting forever — a classic deadlock.

```python
import subprocess

# Replicate: cat large.log | grep ERROR | wc -l
# Without shell=True — fully safe, no injection possible

p1 = subprocess.Popen(['cat', 'large.log'],   stdout=subprocess.PIPE)
p2 = subprocess.Popen(['grep', 'ERROR'],       stdin=p1.stdout, stdout=subprocess.PIPE)
p1.stdout.close()   # ← IMPORTANT: allows p1 to receive SIGPIPE when p2 exits
p3 = subprocess.Popen(['wc', '-l'],            stdin=p2.stdout, stdout=subprocess.PIPE)
p2.stdout.close()   # ← same reason

output, _ = p3.communicate()
print(f"Error lines: {output.decode().strip()}")
```

## Part 5: `subprocess.Popen()` — Advanced Control

`subprocess.Popen()` creates a process without blocking. Use it for:
- Running commands in the background while doing other work
- Real-time output streaming
- Long-running processes that need monitoring

### Real-time Output Streaming

```python
import subprocess

def run_with_realtime_output(cmd: list[str]) -> int:
    """Run command and print output line by line as it arrives"""
    process = subprocess.Popen(
        cmd,
        stdout=subprocess.PIPE,
        stderr=subprocess.STDOUT,  # merge stderr into stdout
        text=True,
        bufsize=1    # line-buffered: flush after each \n
    )

    for line in process.stdout:
        print(line, end='')        # print immediately, no buffering

    process.wait()                 # wait for process to finish
    return process.returncode

# Usage: see kubectl rollout status output live
exit_code = run_with_realtime_output(['kubectl', 'rollout', 'status', 'deployment/myapp'])
```

### Background Process

```python
import subprocess
import time

# Start process without waiting
process = subprocess.Popen(['ping', '-c', '100', 'google.com'], stdout=subprocess.PIPE)

# Do other work while ping runs
for i in range(5):
    print(f"Doing other work... {i}")
    time.sleep(1)

    # Check if process finished (non-blocking)
    if process.poll() is not None:
        print(f"Process finished with code {process.returncode}")
        break

# Terminate if still running
if process.poll() is None:
    process.terminate()
    process.wait(timeout=5)   # wait for clean exit
```

## Part 6: Retry Logic for Commands

```python
import subprocess
import time
import logging

logger = logging.getLogger(__name__)

def run_with_retry(
    cmd: list[str],
    retries: int = 3,
    delay: float = 5,
    backoff: float = 2,
    timeout: int = 30
) -> tuple[bool, str, str]:
    """Run command with retry and exponential backoff"""
    current_delay = delay

    for attempt in range(1, retries + 1):
        result = subprocess.run(
            cmd, capture_output=True, text=True, timeout=timeout
        )

        if result.returncode == 0:
            return True, result.stdout, result.stderr

        logger.warning(
            f"Attempt {attempt}/{retries} failed (exit {result.returncode}). "
            f"Retrying in {current_delay:.0f}s..."
        )
        if attempt < retries:
            time.sleep(current_delay)
            current_delay *= backoff

    return False, result.stdout, result.stderr

# Usage: kubectl apply occasionally fails on network blips
success, stdout, stderr = run_with_retry(
    ['kubectl', 'apply', '-f', 'deployment.yaml'],
    retries=5, delay=10, backoff=2
)
```

## Part 7: Practical Examples for Platform Engineers

### Example 1: Check Kubernetes Pod Status

```python
import subprocess
import json
from typing import Optional

def get_pod_status(namespace: str = 'default') -> list[dict]:
    """Get status of all pods in a namespace"""
    cmd = ['kubectl', 'get', 'pods', '-n', namespace, '-o', 'json']
    result = subprocess.run(cmd, capture_output=True, text=True, timeout=30)

    if result.returncode != 0:
        print(f"Error: {result.stderr}")
        return []

    data = json.loads(result.stdout)
    pods = []

    for item in data.get('items', []):
        pods.append({
            'name':     item['metadata']['name'],
            'status':   item['status']['phase'],
            'node':     item['spec'].get('nodeName', 'N/A'),
            'restarts': sum(
                c.get('restartCount', 0)
                for c in item['status'].get('containerStatuses', [])
            )
        })

    return pods

pods = get_pod_status('production')
for pod in pods:
    print(f"{pod['name']:<40} {pod['status']:<12} restarts={pod['restarts']}")
```

### Example 2: Docker Image Cleanup

```python
import subprocess

def delete_old_images(days: int = 30) -> bool:
    """Delete Docker images older than specified days"""
    cmd = ['docker', 'image', 'prune', '--all', '--filter', f'until={days * 24}h', '--force']
    result = subprocess.run(cmd, capture_output=True, text=True, timeout=120)

    if result.returncode == 0:
        print(f"Cleaned up images older than {days} days")
        print(result.stdout)
        return True
    else:
        print(f"Failed: {result.stderr}")
        return False

def get_docker_images() -> list[dict]:
    """Get list of Docker images"""
    result = subprocess.run(
        ['docker', 'images', '--format', '{{.Repository}}:{{.Tag}}\t{{.Size}}'],
        capture_output=True, text=True, timeout=30
    )
    images = []
    for line in result.stdout.strip().split('\n'):
        if line:
            name, size = line.split('\t', 1)
            images.append({'name': name, 'size': size})
    return images
```

### Example 3: System Resource Monitor

```python
import subprocess
import re

def get_disk_usage(path: str = '/') -> Optional[float]:
    """Get disk usage percentage for a path"""
    result = subprocess.run(['df', '-h', path], capture_output=True, text=True)
    lines = result.stdout.strip().split('\n')
    if len(lines) >= 2:
        parts = lines[1].split()
        return float(parts[4].rstrip('%'))  # "85%" → 85.0
    return None

def check_system_health(thresholds: dict | None = None) -> dict[str, bool]:
    """Check system health metrics"""
    thresholds = thresholds or {'disk': 90, 'cpu': 90, 'memory': 90}

    health = {}

    disk = get_disk_usage('/')
    if disk is not None:
        health['disk'] = disk < thresholds['disk']
        if not health['disk']:
            print(f"⚠️  Disk usage critical: {disk:.1f}%")

    return health
```

## Summary Tables

### `subprocess.run()` Parameters

| Parameter | Purpose | Default |
|-----------|---------|---------|
| `capture_output` | Capture stdout+stderr | `False` |
| `text` | Return strings (not bytes) | `False` |
| `check` | Raise on non-zero exit | `False` |
| `timeout` | Timeout in seconds | `None` |
| `shell` | Use shell (`bash -c`) | `False` (avoid) |
| `env` | Custom environment dict | Inherit parent |
| `cwd` | Working directory | Current dir |
| `stdout=subprocess.DEVNULL` | Discard stdout | — |
| `stderr=subprocess.DEVNULL` | Discard stderr | — |
| `stderr=subprocess.STDOUT` | Merge stderr→stdout | — |

### Common Patterns

| Pattern | Code |
|---------|------|
| Run + ignore output | `subprocess.run(['cmd', 'arg'])` |
| Capture output | `r = subprocess.run(['cmd'], capture_output=True, text=True)` |
| Raise on failure | `subprocess.run(['cmd'], check=True)` |
| With timeout | `subprocess.run(['cmd'], timeout=30)` |
| Custom env | `subprocess.run(['cmd'], env={**os.environ, 'KEY': 'val'})` |
| Safe string split | `shlex.split("cmd --flag 'arg with spaces'")` |
| Discard output | `subprocess.run(['cmd'], stdout=subprocess.DEVNULL)` |
| Real-time stream | `Popen(['cmd'], stdout=PIPE, text=True, bufsize=1)` |

### `subprocess.run()` vs `Popen()`

| Feature | `subprocess.run()` | `subprocess.Popen()` |
|---------|-------------------|---------------------|
| Waits for completion | ✅ Always | ❌ No — non-blocking |
| Returns | `CompletedProcess` | `Popen` object |
| Capture output | `capture_output=True` | `stdout=PIPE` |
| Real-time streaming | ❌ | ✅ |
| Background process | ❌ | ✅ |
| Timeout | `timeout=` | `.wait(timeout=)` |
| Use when | Most cases | Streaming / background |
