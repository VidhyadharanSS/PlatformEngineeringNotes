## Cheatsheet

### `subprocess.run()` — Core Pattern

```python
import subprocess

# Capture output (most common)
result = subprocess.run(
    ['kubectl', 'get', 'pods', '-n', 'production'],
    capture_output=True,   # stdout + stderr → PIPE
    text=True,             # decode bytes to str
    timeout=30,            # raise TimeoutExpired after 30s
    check=False            # don't raise on non-zero exit
)

# Use result
if result.returncode == 0:
    print(result.stdout)
else:
    print(f"Error: {result.stderr}", file=sys.stderr)

# Raise on non-zero exit
try:
    subprocess.run(['kubectl', 'apply', '-f', 'deploy.yaml'], check=True,
                   capture_output=True, text=True)
except subprocess.CalledProcessError as e:
    print(f"Failed ({e.returncode}): {e.stderr}")
```

### `subprocess.run()` Parameters

| Parameter | Purpose | Default |
|-----------|---------|---------|
| `capture_output=True` | Capture stdout+stderr into PIPE | `False` |
| `text=True` | Return strings, not bytes | `False` |
| `check=True` | Raise `CalledProcessError` on failure | `False` |
| `timeout=N` | Raise `TimeoutExpired` after N seconds | `None` |
| `shell=True` | Use `bash -c` (avoid with input) | `False` |
| `env=dict` | Custom env vars | Inherit parent |
| `cwd=path` | Working directory | Current dir |
| `stdout=subprocess.DEVNULL` | Discard stdout | — |

### shlex — Safe String Splitting

```python
import shlex

# String from config → safe list for subprocess
cmd  = "kubectl get pods --field-selector 'status.phase=Running'"
args = shlex.split(cmd)    # handles quotes correctly
subprocess.run(args)

# List → display string (for logging)
print(shlex.join(['ls', '-la', '/path with spaces']))
```

### argparse — Quick Setup

```python
import argparse

parser = argparse.ArgumentParser(
    description='My tool',
    formatter_class=argparse.ArgumentDefaultsHelpFormatter  # shows defaults in --help
)

# Positional
parser.add_argument('environment', help='Target env')

# Optional
parser.add_argument('-n', '--namespace', default='default')

```

### pathlib — Modern Path Operations

```python
from pathlib import Path

# Create
p    = Path('/var/log/nginx/access.log')
home = Path.home()
cwd  = Path.cwd()

# Join with /
p2 = Path('/var/log') / 'nginx' / 'access.log'

# Components
p.name      # 'access.log'
p.stem      # 'access'
p.suffix    # '.log'
p.parent    # Path('/var/log/nginx')

# Checks
p.exists()
p.is_file()
p.is_dir()

# Read/Write
content = Path('f.txt').read_text()
Path('f.txt').write_text('content')

# Directory
for f in Path('/var/log').glob('*.log'):     print(f)   # non-recursive
for f in Path('/home').rglob('*.py'):        print(f)   # recursive
Path('a/b/c').mkdir(parents=True, exist_ok=True)
Path('f.txt').unlink(missing_ok=True)

# Metadata
stat = path.stat()
size_mb = stat.st_size / 1024 / 1024
```

## Comparison Tables

### `subprocess.run()` vs `os.system()`

| Feature | `subprocess.run()` | `os.system()` |
|---------|-------------------|--------------|
| Output capture | ✅ | ❌ |
| Injection risk | Low (list) | High (shell) |
| Return code | `result.returncode` | Limited |
| Timeout | `timeout=N` | ❌ |
| Custom env | `env=dict` | ❌ |
| Recommendation | **Always use** | Never |

### `argparse` vs `sys.argv`

| Feature | `argparse` | `sys.argv` |
|---------|-----------|-----------|
| `--help` auto-generated | ✅ | ❌ |
| Type conversion | ✅ | Manual |
| Default values | ✅ | Manual |
| Choices validation | ✅ | Manual |
| Subcommands | ✅ | Manual |
| Recommendation | **Always use** | Simple scripts only |

### `pathlib` vs `os.path`

| Feature | `pathlib` | `os.path` |
|---------|-----------|---------|
| Object-oriented | ✅ | ❌ |
| Join with `/` operator | ✅ | ❌ |
| `read_text()` / `write_text()` | ✅ | ❌ |
| `rglob()` built-in | ✅ | ❌ |
| Works with `open()` | ✅ | ✅ |
| Recommendation | **New code** | Legacy code |

## Bridge Concepts

| Concept | Explanation |
|---------|-------------|
| `shlex.split()` | Splits a shell command string into a safe list respecting quotes. `shlex.split("cmd 'a b'")` → `['cmd', 'a b']` |
| `subprocess.DEVNULL` | Equivalent to redirecting to `/dev/null`. Use when you want a command to run silently. |
| `subprocess.STDOUT` | Redirect stderr to stdout stream. `stderr=subprocess.STDOUT` merges error output. |
| `bufsize=1` | Line-buffered output. Needed for real-time streaming with `Popen`. |
| `p1.stdout.close()` | In pipe chains, closes the write end so `p2` receives EOF. Without this, `p2` hangs forever waiting for more input. |
| `capture_output=True` | Shorthand for `stdout=subprocess.PIPE, stderr=subprocess.PIPE`. Added Python 3.7. |
| `python-dotenv` | Third-party library. `pip install python-dotenv`. Loads `.env` file into `os.environ`. Never use in production containers. |
| `nargs='+'` | argparse: one or more arguments → list. `nargs='*'` → zero or more. `nargs='?'` → zero or one. |
| `action='count'` | argparse: count how many times flag appears. `-v` → 1, `-vv` → 2. |
| `formatter_class=ArgumentDefaultsHelpFormatter` | Shows default values in `--help` output automatically. |
| `Path.relative_to()` | Compute relative path between two absolute paths. Used when copying directory trees. |
| `Path.stat()` | Returns file metadata: `st_size` (bytes), `st_mtime` (modified time), `st_mode` (permissions). |
| `Path.expanduser()` | Expands `~` to the home directory. Use when paths come from user input or env vars. |

## Interview Questions

### Question 1

**Scenario:** A script calls `kubectl rollout status deployment/myapp` which may take up to 5 minutes. If it times out or fails, the CI pipeline should fail with a clear error.

**Question:** Write a function with proper timeout, error handling, and logging.

**Answer:**

```python
import subprocess
import sys
import logging

logger = logging.getLogger(__name__)

def wait_for_rollout(deployment: str, namespace: str = 'default', timeout: int = 300) -> bool:
    """Wait for Kubernetes rollout to complete"""
    cmd = ['kubectl', 'rollout', 'status', f'deployment/{deployment}',
           '-n', namespace, f'--timeout={timeout}s']

    logger.info(f"Waiting for {deployment} rollout (timeout={timeout}s)")

    try:
        result = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            timeout=timeout + 10   # slightly longer than kubectl's own timeout
        )
        if result.returncode == 0:
            logger.info(f"✅ Rollout successful: {deployment}")
            return True
        else:
            logger.error(f"❌ Rollout failed: {result.stderr}")
            return False

    except subprocess.TimeoutExpired:
        logger.error(f"⏰ Python timeout after {timeout + 10}s")
        return False
    except FileNotFoundError:
        logger.error("kubectl not found in PATH")
        return False

if not wait_for_rollout('myapp', 'production', timeout=300):
    sys.exit(1)
```

### Question 2

**Scenario:** A deployment script accepts `--env` (required, one of dev/staging/prod) and `--image` (required). It should load env-specific config from env vars like `DEV_DB_HOST`, `PROD_DB_HOST`.

**Question:** Write the argument parsing and config loading.

**Answer:**

```python
import argparse
import os
import sys

def parse_args() -> argparse.Namespace:
    parser = argparse.ArgumentParser(
        description='Deploy application',
        formatter_class=argparse.ArgumentDefaultsHelpFormatter
    )
    parser.add_argument('--env',   required=True, choices=['dev', 'staging', 'prod'])
    parser.add_argument('--image', required=True, help='Docker image:tag')
    parser.add_argument('-v', '--verbose', action='store_true')
    parser.add_argument('--dry-run', action='store_true')
    return parser.parse_args()

def load_env_config(env: str) -> dict:
    prefix = env.upper()  # DEV_, STAGING_, PROD_
    config = {
        'db_host':   os.environ.get(f'{prefix}_DB_HOST', 'localhost'),
        'db_port':   int(os.environ.get(f'{prefix}_DB_PORT', '5432')),
        'namespace': os.environ.get(f'{prefix}_NAMESPACE', env),
        'replicas':  int(os.environ.get(f'{prefix}_REPLICAS', '1')),
    }

    # Required vars
    api_key = os.environ.get(f'{prefix}_API_KEY')
    if not api_key:
        raise ValueError(f"{prefix}_API_KEY environment variable required")
    config['api_key'] = api_key

    return config

def main() -> int:
    args = parse_args()
    try:
        config = load_env_config(args.env)
    except ValueError as e:
        print(f"Configuration error: {e}", file=sys.stderr)
        return 1

    print(f"Deploying {args.image} to {args.env} (ns={config['namespace']}, replicas={config['replicas']})")
    if args.dry_run:
        print("[DRY RUN] No changes made")
        return 0
    # ... actual deployment
    return 0

if __name__ == '__main__':
    sys.exit(main())
```

### Question 3

**Scenario:** A script needs to find all Kubernetes YAML manifests in a repo and validate them with `kubectl apply --dry-run=client`. Collect errors, report at the end.

**Question:** Use `pathlib` for file discovery and `subprocess` for validation.

**Answer:**

```python
import subprocess
from pathlib import Path
import sys

def validate_manifests(directory: str) -> bool:
    """Find and validate all K8s YAML files"""
    root    = Path(directory)
    skip    = {'.git', 'node_modules', 'vendor'}
    errors  = []
    checked = 0

    manifests = [
        f for f in root.rglob('*.yaml')
        if not any(part in skip for part in f.parts)
    ] + [
        f for f in root.rglob('*.yml')
        if not any(part in skip for part in f.parts)
    ]

    print(f"Found {len(manifests)} manifest files")

    for manifest in sorted(manifests):
        checked += 1
        try:
            result = subprocess.run(
                ['kubectl', 'apply', '--dry-run=client', '-f', str(manifest)],
                capture_output=True, text=True, timeout=30
            )
            if result.returncode == 0:
                print(f"  ✅ {manifest.relative_to(root)}")
            else:
                msg = f"  ❌ {manifest.relative_to(root)}: {result.stderr.strip()}"
                errors.append(msg)
                print(msg)
        except subprocess.TimeoutExpired:
            msg = f"  ⏰ {manifest.relative_to(root)}: timeout"
            errors.append(msg)
            print(msg)

    print(f"\nSummary: {checked} checked, {len(errors)} errors")
    return len(errors) == 0

if __name__ == '__main__':
    directory = sys.argv[1] if len(sys.argv) > 1 else '.'
    sys.exit(0 if validate_manifests(directory) else 1)
```

### Question 4

**Scenario:** A script must run on both developer laptops (reading from `.env`) and in Kubernetes pods (env vars injected by the container runtime). Write a config loader that handles both cases transparently.

**Answer:**

```python
import os
from pathlib import Path

def load_env_if_dev() -> None:
    """Load .env file only in development (not when running in Kubernetes)"""
    # Kubernetes injects KUBERNETES_SERVICE_HOST
    if os.environ.get('KUBERNETES_SERVICE_HOST'):
        return   # running in Kubernetes — env vars already injected

    # Look for .env file
    env_file = Path.cwd() / '.env'
    if not env_file.exists():
        env_file = Path.home() / '.config' / 'myapp' / '.env'

    if env_file.exists():
        try:
            from dotenv import load_dotenv
            load_dotenv(env_file)
            print(f"Loaded .env from {env_file}")
        except ImportError:
            # python-dotenv not installed — manual load
            with open(env_file) as f:
                for line in f:
                    line = line.strip()
                    if line and not line.startswith('#') and '=' in line:
                        k, _, v = line.partition('=')
                        os.environ.setdefault(k.strip(), v.strip())

class Config:
    def __init__(self):
        load_env_if_dev()   # no-op in production
        self.db_host  = os.environ.get('DB_HOST', 'localhost')
        self.db_port  = int(os.environ.get('DB_PORT', '5432'))
        self.api_key  = self._require('API_KEY')
        self.env      = os.environ.get('ENVIRONMENT', 'dev')

    def _require(self, key: str) -> str:
        val = os.environ.get(key)
        if not val:
            raise ValueError(f"Required environment variable {key!r} is not set")
        return val

# Usage works in both dev (reads .env) and prod (reads K8s env vars)
config = Config()
```
